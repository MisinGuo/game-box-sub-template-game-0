# 路由管理系统

## 概述

统一管理所有页面的路由、导航、SEO、sitemap等配置，一处修改，全站生效。

## 核心文件

### 📋 `src/config/site/routes.ts`
**统一路由配置文件** - 所有页面配置的唯一真实来源

```typescript
{
  path: '/games',              // 路由路径
  title: '游戏库',             // 页面标题
  description: '...',          // 页面描述
  enabled: true,               // 是否启用
  showInNav: true,             // 是否在导航显示
  navOrder: 2,                 // 导航顺序
  inSitemap: true,             // 是否在sitemap
  sitemapPriority: 0.9,        // sitemap优先级
  sitemapChangeFreq: 'daily',  // 更新频率
  parent: 'guides',            // 父级菜单（可选）
}
```

### 🌐 `src/config/site/site.ts`
**站点配置** - 自动从 routes.ts 生成导航

```typescript
export const siteConfig: SiteConfig = {
  nav: buildNavMenu(), // 自动生成导航菜单
  // ...其他配置
}
```

### 🗺️ `src/app/sitemap.ts`
**自动生成 sitemap.xml** - 从 routes.ts 读取配置

### 🤖 `src/app/robots.ts`
**自动生成 robots.txt** - 包含 sitemap 链接

## 使用场景

### 1️⃣ 添加新页面

```typescript
// 在 routes.ts 中添加配置
{
  path: '/new-page',
  title: '新页面',
  enabled: true,        // ✅ 启用
  showInNav: true,      // ✅ 显示在导航
  navOrder: 6,
  inSitemap: true,      // ✅ 包含在sitemap
  sitemapPriority: 0.8,
}
```

**自动生效：**
- ✅ 导航栏显示"新页面"
- ✅ sitemap.xml 包含 `/new-page`
- ✅ 按 navOrder 排序

### 2️⃣ 隐藏页面

```typescript
// 只需修改一处
{
  path: '/special',
  title: '特价游戏',
  enabled: false,       // ❌ 禁用
  showInNav: false,     // ❌ 不显示在导航
  inSitemap: false,     // ❌ 不包含在sitemap
}
```

**自动生效：**
- ❌ 导航栏不显示
- ❌ sitemap.xml 不包含
- ✅ 页面仍可访问（如需完全移除，删除对应目录）

### 3️⃣ 创建下拉菜单

```typescript
// 父菜单项
{
  path: '/guides',
  title: '攻略中心',
  showInNav: true,
  navOrder: 4,
}

// 子菜单项
{
  path: '/strategy',
  title: '游戏攻略',
  parent: 'guides',     // 🔗 指向父级
  showInNav: true,
  navOrder: 5,
}
```

**生成结果：**
```
攻略中心 ▼
  ├─ 游戏攻略
  └─ (其他子项)
```

### 4️⃣ SEO优化

```typescript
{
  path: '/',
  title: '首页',
  inSitemap: true,
  sitemapPriority: 1.0,        // 🔥 最高优先级
  sitemapChangeFreq: 'daily',  // 📅 每日更新
}

{
  path: '/about',
  title: '关于',
  inSitemap: true,
  sitemapPriority: 0.3,        // 低优先级
  sitemapChangeFreq: 'monthly', // 月度更新
}
```

## 工具函数

### `getEnabledRoutes()`
获取所有启用的路由

```typescript
const routes = getEnabledRoutes()
// 返回所有 enabled: true 的路由
```

### `getNavRoutes()`
获取导航栏路由（已排序）

```typescript
const navRoutes = getNavRoutes()
// 返回 enabled && showInNav 的路由，按 navOrder 排序
```

### `getSitemapRoutes()`
获取sitemap路由

```typescript
const sitemapRoutes = getSitemapRoutes()
// 返回 enabled && inSitemap 的路由
```

### `buildNavMenu()`
构建导航菜单（支持嵌套）

```typescript
const menu = buildNavMenu()
// 返回导航菜单结构，自动处理父子关系
```

### `getRouteByPath(path)`
根据路径获取配置

```typescript
const config = getRouteByPath('/games')
// 返回 /games 的完整配置
```

## 最佳实践

### ✅ DO

1. **单一真实来源** - 只在 `routes.ts` 中配置路由
2. **语义化命名** - path、title 保持一致
3. **合理优先级** - 首页1.0，核心页面0.8-0.9，普通页面0.5-0.7
4. **更新频率** - 根据实际更新频率设置 changeFreq
5. **有序组织** - 使用 navOrder 控制显示顺序

### ❌ DON'T

1. **不要在多处配置** - 避免在组件中硬编码导航
2. **不要忘记更新** - 添加新页面时记得在 routes.ts 添加配置
3. **不要过度嵌套** - 菜单层级最多2层
4. **不要随意优先级** - sitemap 优先级需要合理规划

## 扩展功能

### 添加面包屑导航

```typescript
export function getBreadcrumb(path: string) {
  const route = getRouteByPath(path)
  const crumbs = [{ text: '首页', link: '/' }]
  
  if (route?.parent) {
    const parent = routes.find(r => r.path === route.parent)
    if (parent) crumbs.push({ text: parent.title, link: parent.path })
  }
  
  if (route) crumbs.push({ text: route.title, link: route.path })
  
  return crumbs
}
```

## 常见问题

### Q: 如何临时禁用页面？
A: 设置 `enabled: false`，页面仍可访问但不显示在导航和sitemap

### Q: 如何完全移除页面？
A: 
1. 设置 `enabled: false`
2. 删除对应页面目录（如 `app/[locale]/special/`）
3. （可选）添加重定向规则

### Q: sitemap 何时生成？
A: Next.js 构建时自动生成，访问 `/sitemap.xml` 查看

### Q: 如何测试导航？
A: 运行开发服务器，检查导航栏是否正确显示

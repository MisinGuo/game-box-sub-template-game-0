# 配置系统总览

## 📂 配置目录结构

```
src/config/
├── index.ts                  # 统一导出入口
├── types.ts                  # 全局类型定义
│
├── site/                     # 站点配置 🌐
│   ├── index.ts              # 站点配置导出
│   ├── site.ts               # 基本信息（名称、域名、Logo等）
│   ├── theme.ts              # 主题样式（颜色、字体、布局等）
│   ├── locales.ts            # 多语言翻译文本
│   ├── modules.ts            # 功能模块配置
│   └── routes.ts             # 导航路由配置
│
├── api/                      # API配置 🔌
│   ├── index.ts              # API配置导出
│   └── backend.ts            # 后端接口配置（读取环境变量）
│
├── pages/                    # 页面配置 📄
│   ├── index.ts              # 页面配置导出
│   ├── home.ts               # 首页配置 (/)
│   ├── games.ts              # 游戏库配置 (/games)
│   ├── boxes.ts              # 盒子大全配置 (/boxes)
│   ├── content.ts            # 内容中心配置 (/content)
│   ├── news.ts               # 资讯配置 (/news)
│   └── search.ts             # 搜索页配置 (/search)
│
└── sitemap/                  # Sitemap配置
    └── config.ts             # 允许域名白名单、默认域名等
```

## 🎯 配置分类

### 1. 站点配置 (`site/`) 🌐

**用途：** 全站通用的基础配置

**包含文件：**
- `site.ts` - 网站名称、描述、Logo、版权信息等
- `theme.ts` - 颜色、字体、布局等样式配置
- `locales.ts` - 多语言翻译文本（zh-CN / zh-TW / en-US）
- `modules.ts` - 功能模块配置（special、content 等模块）
- `routes.ts` - 导航菜单路由配置

**修改频率：** 低（项目初始化后很少改动）

**使用示例:**
```typescript
import { siteConfig } from '@/config'

console.log(siteConfig.name)     // 网站名称
console.log(siteConfig.hostname) // 网站域名
```

### 2. API配置 (`api/`) 🔌

**用途：** 后端接口配置，从环境变量读取

**包含文件：**
- `backend.ts` - API 基础地址、超时时间、站点 ID 等

**关键配置（通过环境变量注入）：**
```typescript
// src/config/api/backend.ts
export const backendConfig: BackendConfig = {
  baseURL: process.env.NEXT_PUBLIC_API_URL || 'http://localhost:8080',
  siteId: process.env.NEXT_PUBLIC_SITE_ID || '1',
  apiKey: process.env.NEXT_PUBLIC_API_KEY || '',
  timeout: 10000,
}
```

**修改频率：** 通过 `.env.local` 或 Cloudflare 环境变量修改，无需改代码

### 3. 页面配置 (`pages/`) 📄

**用途：** 各页面的 Hero、Section、SEO 等 UI 配置

**包含文件：**
- `home.ts` - 首页 Hero、统计数字、内容区块
- `games.ts` - 游戏库页面配置
- `boxes.ts` - 盒子大全页面配置
- `content.ts` - 内容中心（攻略、评测、专题）配置
- `news.ts` - 资讯列表配置
- `search.ts` - 搜索页配置

**使用示例:**
```typescript
import { getHomePageConfig } from '@/config'

const config = getHomePageConfig('zh-CN')
// config.hero.title, config.sections, etc.
```

### 4. Sitemap 配置 (`sitemap/`)

**用途：** 控制 Sitemap 生成行为

**关键配置：**
```typescript
// src/config/sitemap/config.ts
export const sitemapConfig = {
  allowedHosts: ['www.5awyx.com', 'star-origin.5awyx.com', ...],
  defaultHostname: 'https://www.5awyx.com',  // 域名校验失败时的 fallback
  maxUrlsPerSitemap: 50000,
}
```

安全机制：通过 `X-Forwarded-Host` 请求头识别真实访问域名，防止内容被盗用。

## 📋 配置清单

启动新项目时需要修改的配置：

- [ ] `site/site.ts` - 网站名称、域名、Logo
- [ ] `api/backend.ts` - API 地址（通过 `.env.local` 注入）
- [ ] `pages/home.ts` - 首页内容
- [ ] `site/theme.ts` - 主题颜色（可选）
- [ ] `.env.local` - 本地环境变量

## 🐛 常见问题

### Q: 配置修改后不生效？

1. 检查是否正确导入配置
2. 清除 Next.js 缓存（删除 `.next` 目录）
3. 重启开发服务器

### Q: TypeScript 报错？

1. 确认配置类型正确
2. 运行 `pnpm type-check`
3. 查看 `types.ts` 中的类型定义

# 类型定义组织规范

## 📁 目录结构

```
src/config/
├── types.ts                  # 核心类型（Locale、ModuleType、ThemeConfig 等）
├── types/                    # 共享组件类型
│   ├── common.ts            # 通用 UI 类型（LocalizedString、SeoMetadata、LinkConfig 等）
│   └── index.ts             # 统一导出
├── pages/                   # 页面配置（扁平文件结构）
│   ├── index.ts             # 导出所有页面配置
│   ├── home.ts              # 首页配置 (/)
│   ├── games.ts             # 游戏库配置 (/games)
│   ├── boxes.ts             # 盒子大全配置 (/boxes)
│   ├── content.ts           # 内容中心配置 (/content)
│   ├── news.ts              # 资讯配置 (/news)
│   └── search.ts            # 搜索页配置 (/search)
└── site/                    # 站点配置
    ├── site.ts              # 站点基础信息
    ├── theme.ts             # 主题样式
    ├── locales.ts           # 多语言配置
    ├── modules.ts           # 功能模块配置
    ├── routes.ts            # 导航路由配置
    └── index.ts             # 统一导出
```

## ✅ 设计原则

### 1. 两层类型分离

项目将类型分为两层：

- **核心类型** (`types.ts`)：`Locale`、`ModuleType`、`ThemeConfig`、`LocaleConfig`、`SidebarConfig` 等项目级基础类型
- **通用 UI 类型** (`types/common.ts`)：可复用的 UI 组件类型，如 `LocalizedString`、`SeoMetadata`、`LinkConfig`、`ButtonConfig`、`LayoutType`

```typescript
// config/types.ts — 项目核心类型
export type Locale = 'zh-CN' | 'zh-TW' | 'en-US'
export interface ThemeConfig { ... }
export interface LocaleConfig { ... }

// config/types/common.ts — 通用 UI 类型
import type { Locale } from '../types'
export type LocalizedString = Record<Locale, string>
export type LocalizedArray<T> = Record<Locale, T[]>
export interface SeoMetadata { title: string; description: string; keywords?: string[] }
export interface LinkConfig { text: string; href: string }
export interface ButtonConfig extends LinkConfig { variant?: string }
export type LayoutType = 'grid-2' | 'grid-3' | 'grid-4' | 'grid-6' | 'list' | 'compact'
```

### 2. 页面配置扁平化

页面配置采用**扁平文件结构**，每个路由对应一个配置文件，类型直接内联在文件中，无需子目录：

```typescript
// ✅ 实际结构：扁平文件
// config/pages/games.ts — 类型和配置写在一起
export interface GamesPageConfig { hero: { title: string; description: string } }

export function getGamesPageConfig(locale: string): GamesPageConfig {
  return { ... }
}
```

**优势：**
- 路由与配置文件一一对应，易于查找
- 减少不必要的目录层级，结构简洁
- 小规模配置无需拆分 types.ts

### 3. 类型复用（Type Reuse）
**通用类型从 `types/common.ts` 导入**

```typescript
// config/pages/home.ts
import type { LocalizedString, SeoMetadata, ButtonConfig } from '../types/common'

export interface HomeHeroConfig {
  title: LocalizedString   // ✅ 复用通用类型
  primaryButton: ButtonConfig
}
```

### 4. 清晰的依赖层级（Clear Dependencies）
```
config/types.ts           (最底层：Locale、ThemeConfig 等)
    ↓
config/types/common.ts    (通用 UI 类型，依赖 types.ts 的 Locale)
    ↓
config/pages/games.ts     (页面配置，依赖 types/common)
    ↓
app/games/page.tsx        (使用页面配置)
```

## 📝 命名规范

### 类型命名
- **接口**：使用 PascalCase + Config/Item/Options 后缀
  ```typescript
  interface HomePageConfig { }
  interface StatItem { }
  interface FilterOptions { }
  ```

- **类型别名**：使用 PascalCase
  ```typescript
  type LocalizedString = string | Record<string, string>
  type LayoutType = 'grid-3' | 'grid-4' | 'grid-6'
  ```

### 文件命名
- **类型文件**：`types.ts`
- **配置文件**：对应功能名，如 `home.ts`, `list.ts`
- **导出文件**：`index.ts`

## 🔍 导入规范

### 推荐的导入方式
```typescript
// ✅ 从配置入口导入
import type { Locale } from '@/config/types'
import type { LocalizedString, SeoMetadata } from '@/config/types/common'

// ✅ 从页面配置导入（类型内联在配置文件中）
import { getGamesPageConfig } from '@/config/pages/games'
import type { GamesPageConfig } from '@/config/pages/games'

// ✅ 从配置统一入口导入
import { siteConfig } from '@/config'
```

## 🎯 实战示例

### 添加新页面配置

1. **在 `pages/` 下创建配置文件**
```typescript
// src/config/pages/topics.ts
import type { SeoMetadata } from '../types/common'

export interface TopicsPageConfig {
  hero: {
    title: string
    description: string
  }
  metadata: SeoMetadata
}

export function getTopicsPageConfig(locale: string): TopicsPageConfig {
  return {
    hero: {
      title: locale === 'en-US' ? 'Topics' : '专题',
      description: '...',
    },
    metadata: {
      title: '专题 | GameBox',
      description: '...',
    },
  }
}
```

2. **在 `pages/index.ts` 中导出**
```typescript
export { getTopicsPageConfig } from './topics'
export type { TopicsPageConfig } from './topics'
```

3. **在页面组件中使用**
```typescript
// app/content/topics/page.tsx
import { getTopicsPageConfig } from '@/config/pages/topics'

export default function TopicsPage({ params }: { params: { locale: string } }) {
  const config = getTopicsPageConfig(params.locale)
  return <h1>{config.hero.title}</h1>
}
```

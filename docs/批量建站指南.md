# 批量建站使用指南

> 基于 `Next-web-sub` 模板，适用于快速复制新垂直内容站。
> 核心原则：**代码可完全共用，内容必须各站独立**。

---

## 一、为什么可以用同一套代码

Google 判定重复内容看的是 **渲染后的 HTML 文本**（title、meta description、h1、正文），而不是代码结构。只要每个站的文案和文章内容不同，使用相同模板框架完全没有问题。

---

## 二、每建一个新站需要改什么

### 第一层：后端（必须）

在管理后台新建一个独立的 **site_id**，并为该站录入专属的文章内容（攻略、评测、资讯）。

> ⚠️ **最高风险点**：多个站点指向同一 `NEXT_PUBLIC_SITE_ID`，会导致所有站展示完全相同的文章，是 SEO 重复内容的最直接来源。

---

### 第二层：环境变量（必须）

在 Cloudflare Pages 项目设置 → Environment variables 中配置以下变量：

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `NEXT_PUBLIC_API_URL` | 后端接口地址 | `https://api.yourdomain.com` |
| `NEXT_PUBLIC_SITE_ID` | **新站专属 ID**（最重要） | `9` |
| `NEXT_PUBLIC_SITE_HOSTNAME` | 新站域名（覆盖 site.ts 里的 hostname） | `https://new-site.com` |
| `NEXT_PUBLIC_MAIN_SITE_URL` | 主站地址（导流目标） | `https://www.5awyx.com` |
| `REVALIDATE_SECRET` | ISR 密钥，每站生成不同的随机字符串 | `openssl rand -hex 32` |
| `NEXT_PUBLIC_GA_ID` | 每站用独立的 GA 账号/属性 | `G-XXXXXXXX` |

---

### 第三层：`src/config/customize/site.ts`（必须）

这是最需要修改的配置文件，决定了站点的**品牌身份**。

```
复制 Next-web-sub 整个仓库后，只需修改这一个文件（加上下方的 fallback-metadata.ts）
即可完成站点差异化，无需动任何组件代码。
```

**必改字段：**

```ts
branding: {
  siteName: '新站名称',                    // 出现在 title、og:title、footer
  siteNameI18n: { 'zh-TW': '繁體站名' },
  tagline: '站点专属定位语',
  description: '站点 SEO 描述，包含核心关键词',
  copyright: 'Copyright © 2025-至今 新站名称',
  logo: '/logo.png',                       // 换 public/logo.png
  ogImage: '/logo.png',
  author: { name: '新站名称', url: 'https://new-site.com', email: '' },
  keywords: ['关键词1', '关键词2', '关键词3'],
  footerDescription: '页脚描述第一行',
  footerDescription2: '页脚描述第二行',
  social: { discord: '', telegram: '' },  // 填新站的或留空
},

site: {
  hostname: 'https://new-site.com',        // 会被 NEXT_PUBLIC_SITE_HOSTNAME 覆盖
  mainSiteUrl: 'https://www.5awyx.com',
  jumpDomain: 'https://www.5awyx.com',
  defaultLocale: 'zh-CN',
  supportedLocales: ['zh-CN', 'zh-TW'],
},

theme: {
  primaryColor: '#3b82f6',                 // 换一个配色，避免视觉相似
  colorScheme: 'blue',                     // purple | blue | green | orange
},

integrations: {
  googleAnalyticsId: 'G-YYYYYY',          // 每站独立 GA 属性
  clarityId: '',
},
```

---

### 第四层：`src/config/customize/fallback-metadata.ts`（必须）

所有列表页（/games、/boxes、/guides、/reviews 等）的 meta title 和 description 都来自这里。**必须重写为新站垂直主题的文案**。

重点修改：

```ts
// 1. 首页 fallback
home: {
  title: { 'zh-CN': '新站名 - 专属 title', ... },
  description: { 'zh-CN': '新站专属描述...', ... },
},

// 2. 各列表页 titleSuffix / descPrefix / featuredContent
metadataTemplates: {
  'zh-CN': {
    games: { titleSuffix: ' - 新主题游戏大全', ... },
    ...
  }
}
```

---

### 第五层：`src/config/customize/pages/home.ts`（必须）

首页 Hero 区域的所有展示文案。Google 爬虫落地首页时看到的 h1/副标题全来自这里。

```ts
hero: {
  badge: { 'zh-CN': '新站专属角标文案' },
  title: { 'zh-CN': '新站主标题' },
  highlightText: { 'zh-CN': '新站高亮副标题' },
  description: { 'zh-CN': ['描述第一行', '描述第二行'] },
},
stats: [
  { label: { 'zh-CN': '关联数字标签' }, value: '200+' },
  ...
],
```

---

### 第六层：`public/` 静态资源（建议）

| 文件 | 说明 |
|------|------|
| `public/logo.png` | 换新站 logo，影响 og:image 和导航栏展示 |
| `public/favicon.ico` | 换新站图标 |
| `public/images/default-cover.svg` | Hero 背景图（可选） |

> og:image 相同的站点在社交分享时会被关联，建议各站使用不同图片。

---

## 三、不需要改的东西

下列内容无需改动，直接继承即可：

- 所有 `src/components/` 下的 React 组件
- `src/lib/` 下的 API 逻辑
- `src/app/` 下的路由结构
- `tailwind.config.js`、`next.config.js`
- SEO 基础设施（JSON-LD、hreflang、sitemap、robots.txt）
- `wrangler.toml`（仅需在 Cloudflare 里改项目名）

## 四、新站建立检查清单

```
后端
  [ ] 后台新建 site，记录 site_id
  [ ] 为该站录入 ≥10 篇原创文章（攻略/评测/资讯）
  [ ] 确认文章与其他站文章不重复

代码仓库
  [ ] Fork / 复制 Next-web-sub 仓库，新命名（如 Next-web-sub-2）
  [ ] 修改 customize/site.ts（品牌、域名、主题色）
  [ ] 修改 customize/fallback-metadata.ts（所有列表页 meta 文案）
  [ ] 修改 customize/pages/home.ts（首页 Hero 文案）
  [ ] 替换 public/logo.png 和 favicon.ico
  [ ] 替换 public/images/default-cover.svg（可选）

Cloudflare Pages
  [ ] 新建 Pages 项目，连接新仓库
  [ ] 配置 NEXT_PUBLIC_SITE_ID（新值，不重复）
  [ ] 配置 NEXT_PUBLIC_SITE_HOSTNAME（新域名）
  [ ] 配置 REVALIDATE_SECRET（独立随机值）
  [ ] 配置 NEXT_PUBLIC_GA_ID（独立 GA 属性）
  [ ] 绑定新域名

上线验证
  [ ] 访问首页，确认 title 包含新站名（不是"卡牌游戏攻略站"）
  [ ] 查看源码，确认 canonical 指向新站域名
  [ ] 访问 /sitemap.xml，确认 URL 都是新站域名
  [ ] 访问 /robots.txt，确认 Sitemap 行指向新站域名
  [ ] 用 https://search.google.com/test/rich-results 验证 JSON-LD
  [ ] 提交新站到 Google Search Console
```

## 五、站点间 SEO 隔离原则

| 维度 | 要求 | 风险等级 |
|------|------|---------|
| 文章内容 | 各站独立，不复制粘贴 | 🔴 致命 |
| site_id | 每站唯一 | 🔴 致命 |
| 域名 | 各站独立域名或子域名 | 🔴 必须 |
| title / meta description | 完全不同的文案 | 🔴 必须 |
| GA 统计 | 各站独立属性 | 🟡 建议 |
| 主题色 / logo | 各站不同 | 🟡 建议 |
| og:image | 各站不同图片 | 🟡 建议 |
| Clarity / 热力图 | 各站独立账号 | 🟢 可选 |
| 代码框架 | 可以相同 | ✅ 无风险 |
| 组件结构 / CSS | 可以相同 | ✅ 无风险 |

---

## 六、关于站点间相互链接

子站之间**不建议**互相添加链接，避免形成明显的链接网络被 Google 识别为 PBN（私人博客网络）。

正确的链接策略：
- 子站 → 主站：✅ 鼓励，帮助主站积累权重
- 主站 → 各子站：✅ 合理，通过后台 API 数据自然关联
- 子站 → 子站：⚠️ 避免，尤其是同主题站之间交叉链接

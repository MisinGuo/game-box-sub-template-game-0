## 一、先纠正口径

### 当前仓位到底代表什么

- `当前仓位` 只代表当天的主主题入口和主打词路。
- 它不代表当天 5 篇都只能围绕同一个游戏写。

```text
3 篇日常池 + 1 篇增量池 + 1 篇测试位
登记表 / 单站排期确认
→ M1 当天游戏池分配
→（缺素材卡的游戏才补跑 M2 / M3 / M4 / M5）
→ M6 今日 5 篇任务筛选
→ M8 正文生成
→ M9 Meta Description
→ M10 后台发布
→ M11 追踪矩阵更新
```

### 人工只负责的部分

1. 用 M7 补真实 `game_id / box_id / 折扣信息`
2. 校验 M2 / M3 / M4 是否跑偏
4. 按 M10 发布并按 M11 回写

## 二、每天的统一执行模板

### 上午：先让 M1 和 M6 决定今天写哪 5 篇

1. 查 [候选子站登记表](../seo/候选子站登记表.md) 和 [site-card-a-7slots](../seo/site-card-a-7slots.md)，确认今天的 `slot / phase / 页面配比`。
2. 用 M7 取今天涉及到的 `日常池 / 增量池 / 测试位` 游戏真实卡片。
3. 跑 M1，得到今天的游戏池分配。
4. 如果今天的增量池或测试位游戏还没有素材卡，再补跑：
   - M2：关键词表
   - M3：痛点清单
   - M5：任务卡矩阵
5. 汇总今天涉及游戏的 M5 任务卡，跑 M6，筛出最终 5 篇。

### 下午：只按 M6 的 5 条任务生成和发布

1. 用 M8 逐篇生成 5 篇正文。
2. 用 M9 生成 5 条描述。
3. 人工替换折扣、链接、图片命名、真实游戏信息。
4. 按 M10 发布。
5. 按 M11 更新去重索引、统计表和当天记录。

### 每天必须留存的工件

- M1 当天游戏池表
- M6 今日 5 篇任务表
- 5 篇 M8 正文 + 5 条 M9 描述
- M11 回写结果

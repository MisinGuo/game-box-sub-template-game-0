# Next-web 快速配置指南

## 🚀 3步完成后端接入

### 步骤1：获取站点ID和API地址

1. 登录游戏推广管理后台
2. 进入 **站点管理**
3. 找到你的站点，记下：
   - 站点ID（如：`1`）
   - 站点域名（如：`https://gamebox.example.com`）
4. 后端API地址通常为：`https://api.example.com`

### 步骤2：配置环境变量

#### 方式A：通过文件配置（本地开发）

1. 复制 `.env.example` 为 `.env.local`
2. 修改配置：

```bash
# 修改为你的后端地址
NEXT_PUBLIC_API_URL=https://api.example.com

# 修改为你的站点ID
NEXT_PUBLIC_SITE_ID=1

# 可选：API密钥
NEXT_PUBLIC_API_KEY=your-api-key
```

#### 方式B：通过Cloudflare Pages配置（生产环境）

在Cloudflare Pages项目设置中添加环境变量：
- `NEXT_PUBLIC_API_URL` = `https://api.example.com`
- `NEXT_PUBLIC_SITE_ID` = `1`
- `NEXT_PUBLIC_API_KEY` = `your-api-key`（可选）

### 步骤3：自定义站点配置

修改 `src/config/site/site.ts`：

```typescript
export const siteConfig: SiteConfig = {
  name: '我的游戏盒子',                    // 🔧 改成你的站点名称
  description: '最全游戏折扣平台',         // 🔧 改成你的描述
  hostname: 'https://mygame.com',         // 🔧 改成你的域名
  jumpDomain: 'https://download.com',     // 🔧 下载跳转域名
  logo: '/logo.svg',                      // 🔧 你的Logo
  favicon: '/favicon.ico',
  copyright: 'Copyright © 2025 MyGame',   // 🔧 改成你的版权
  author: {
    name: 'MyTeam',                       // 🔧 改成你的团队名
    url: 'https://mygame.com',
  },
}
```

### 添加翻译文本

编辑 `src/config/site/locales.ts`：

```typescript
export const locales: Record<Locale, LocaleConfig> = {
  'zh-CN': {
    name: '简体中文',
    code: 'zh-CN',
    translations: {
      // 导航
      home: '首页',
      boxes: '盒子大全',
      // ... 添加你的翻译
      myNewText: '我的新文本',  // 👈 添加这里
    }
  },
  'zh-TW': {
    translations: {
      myNewText: '我的新文本',  // 👈 繁体版本
    }
  },
  'en-US': {
    translations: {
      myNewText: 'My New Text',  // 👈 英文版本
    }
  }
}
```

### 在组件中使用翻译

```typescript
import { locales } from '@/config/site/locales'

export default function MyComponent({ locale }: { locale: Locale }) {
  const t = locales[locale].translations
  
  return (
    <div>
      <h1>{t.myNewText}</h1>
      <p>{t.readAbout} 5 {t.minutesRead}</p>
    </div>
  )
}
```

## 📡 后端接口说明

### 需要后端提供的公开API

所有接口需支持以下请求头：
- `X-Site-Id`: 站点ID
- `X-Api-Key`: API密钥（可选）

#### 1. 文章接口

```
GET /gamebox/article/public/list
参数：pageNum, pageSize, categoryId, locale, siteId
返回：文章列表

GET /gamebox/article/public/detail
参数：id, locale, siteId
返回：文章详情

GET /gamebox/article/public/search
参数：keyword, pageNum, pageSize, siteId
返回：搜索结果

GET /gamebox/article/public/related
参数：articleId, limit, siteId
返回：相关文章
```

#### 2. 分类接口

```
GET /gamebox/category/public/list
参数：categoryType, parentId, locale, siteId
返回：分类列表

GET /gamebox/category/public/tree
参数：categoryType, locale, siteId
返回：分类树
```

#### 3. 盒子接口

```
GET /gamebox/box/public/list
参数：pageNum, pageSize, categoryId, locale, siteId
返回：盒子列表

GET /gamebox/box/public/detail
参数：id, locale, siteId
返回：盒子详情

POST /gamebox/box/public/compare
Body：{ boxIds: [1,2,3], siteId: 1 }
返回：对比数据
```

#### 4. 游戏接口

```
GET /gamebox/game/public/list
参数：pageNum, pageSize, categoryId, locale, siteId
返回：游戏列表

GET /gamebox/game/public/detail
参数：id, locale, siteId
返回：游戏详情

GET /gamebox/game/public/search
参数：keyword, siteId
返回：搜索结果
```

#### 5. 站点配置接口

```
GET /gamebox/site/public/config
参数：siteId, locale
返回：站点配置信息

GET /gamebox/site/public/locales
参数：siteId
返回：支持的语言列表
```

### 响应格式

所有接口返回统一格式：

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": { ... },
  "total": 100
}
```

## 🔄 使用API客户端

在页面组件中使用：

```typescript
import ApiClient from '@/lib/api'

// 获取文章列表
const { data } = await ApiClient.getArticles({
  pageNum: 1,
  pageSize: 10,
  locale: 'zh-CN'
})

// 获取文章详情
const article = await ApiClient.getArticleDetail(1, 'zh-CN')

// 搜索文章
const results = await ApiClient.searchArticles('游戏')
```

## 🚨 常见问题

### Q1: API请求失败，跨域错误？
A: 需要在后端配置CORS，允许你的域名访问。

### Q2: 站点ID怎么获取？
A: 在管理后台的站点管理页面，每个站点都有一个唯一ID。

### Q3: 如何调试API？
A: 
1. 打开浏览器开发者工具
2. 查看 Network 标签
3. 检查请求URL和响应

### Q4: 环境变量不生效？
A: 
1. 确保文件名是 `.env.local`
4. 重启开发服务器 `pnpm dev`
3. Cloudflare Pages需要在设置中配置

### Q5: 如何切换不同的后端？
A: 只需修改 `.env.local` 中的 `NEXT_PUBLIC_API_URL` 和 `NEXT_PUBLIC_SITE_ID`

## 📦 部署

### Cloudflare Workers（推荐）

```bash
pnpm deploy:cfworkers
```

### Cloudflare Pages

1. 将代码推送到 GitHub
2. 在 Cloudflare Pages 创建项目，连接仓库
3. 配置环境变量（`NEXT_PUBLIC_API_URL`、`NEXT_PUBLIC_SITE_ID`、`NEXT_PUBLIC_API_KEY`）
4. 构建命令：`pnpm build`，输出目录：`.next`

> 详细部署步骤请参考 [部署教程](../部署教程.md)
7. 点击部署

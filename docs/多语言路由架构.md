# Next.js 多语言路由架构

## URL 结构设计（SEO友好）

### 默认语言（简体中文 zh-CN）
无路由前缀，直接访问：
- `/` - 首页
- `/games/` - 游戏库列表
- `/games/[id]` - 游戏详情
- `/games/category/[id]` - 游戏分类
- `/boxes/` - 盒子大全列表
- `/boxes/[id]` - 盒子详情
- `/content/` - 内容中心
- `/content/guides/[slug]` - 攻略详情
- `/content/reviews/[slug]` - 评测详情
- `/content/topics/[slug]` - 专题详情
- `/news/[slug]` - 资讯详情
- `/search/` - 搜索

### 其他语言（繁体中文 zh-TW / 英文 en-US）
带语言前缀：
- `/zh-TW/` - 繁体中文首页
- `/zh-TW/games/` - 繁体中文游戏库
- `/zh-TW/content/guides/[slug]` - 繁体中文攻略
- `/en-US/games/` - English games
- `/en-US/boxes/` - English boxes

## 语言切换机制

### 中间件处理
`middleware.ts` 负责语言路由逻辑：
- 默认语言 `zh-CN` 访问 `/zh-CN/xxx` 时自动重定向到 `/xxx`（去除前缀）
- Sitemap 请求 `/sitemap-zh-TW.xml` 被重写到内部 API `/api/sitemap/zh-TW`

### 服务端（generateStaticParams）
```tsx
// 每个多语言页面预生成静态路径
export async function generateStaticParams() {
  return [
    { locale: 'zh-TW' },
    { locale: 'en-US' },
  ]
}
```

## SEO 优化

### 1. **URL 结构清晰**
- 默认语言（zh-CN）无前缀，URL 更简洁
- 其他语言有明确前缀，易于搜索引擎识别

### 2. **独立页面**
- 每个语言版本都是独立的静态页面
- 每个页面有独立的 metadata 和 SEO 标签

### 3. **语言标记**
```tsx
<html lang={locale}>  // 如 "zh-CN"、"zh-TW"、"en-US"
```

### 4. **多语言 Sitemap**
自动生成分语言的 Sitemap：
- `/sitemap.xml` — 索引，列出所有语言的子 Sitemap
- `/sitemap-zh-CN.xml` — 简体中文所有 URL
- `/sitemap-zh-TW.xml` — 繁体中文所有 URL
- `/sitemap-en-US.xml` — 英文所有 URL

每个语言子 Sitemap 进一步按内容类型分组（games, boxes, static 等）。

### 5. **Hreflang 标签**
每个页面生成 hreflang 替代链接：
```html
<link rel="alternate" hreflang="zh-CN" href="https://www.5awyx.com/games/1" />
<link rel="alternate" hreflang="zh-TW" href="https://www.5awyx.com/zh-TW/games/1" />
<link rel="alternate" hreflang="en-US" href="https://www.5awyx.com/en-US/games/1" />
```

## API 集成

所有 API 调用都携带 `locale` 参数，后端根据此参数返回对应语言的内容：

```tsx
ApiClient.getGames({ locale: 'zh-TW', pageNum: 1, pageSize: 20 })
ApiClient.getArticleDetail(slug, 'en-US')
```

## 配置文件

### `src/config/site/locales.ts`
```tsx
export const defaultLocale: Locale = 'zh-CN'
export const supportedLocales: Locale[] = ['zh-CN', 'zh-TW', 'en-US']

export const locales: Record<Locale, LocaleConfig> = {
  'zh-CN': {
    code: 'zh-CN',
    name: '简体中文',
    routePrefix: '',      // 无前缀
    translations: { ... }
  },
  'zh-TW': {
    code: 'zh-TW',
    name: '繁體中文',
    routePrefix: '/zh-TW',
    translations: { ... }
  },
  'en-US': {
    code: 'en-US',
    name: 'English',
    routePrefix: '/en-US',
    translations: { ... }
  }
}
```

## 优势

1. **SEO 友好**：每个语言都有独立的 URL 和 Sitemap
2. **性能优秀**：静态生成 + ISR 按需重新验证
3. **扩展简单**：在 `supportedLocales` 中添加新语言代码即可
4. **默认语言体验好**：`zh-CN` 无前缀，URL 简洁

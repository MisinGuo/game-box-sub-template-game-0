## 二、每个文件讲什么

### [SEO流量战略.md](./SEO流量战略.md)
**整体战略文件，月看一次。**

回答"我们在做什么"：
- 主站（5awyx.com）vs 子站（kapai-web.94kj.cn）的分工
- 整个站点的 URL 结构和页面类型
- 内链系统：每个页面指向谁、从哪里被指向
- Tag 页系统（流量放大器）

> 适合每月做一次战略对齐，或者有新人加入时介绍背景时看。

---

### [内容发布指南.md](./内容发布指南.md)
**发文操作手册，每次发文前查。**

回答"这篇文章该怎么填"：
- 后台 SITE_ID 和 Section Slug 怎么选（填错文章不展示）
- 主站 10+ 种文章类型的定义、标题模板、内链要求
- 子站 10+ 种文章类型的定义（截流/痛点/充值避坑等）
- 快速发文对照表（一张表覆盖所有情况）
- 发布前检查清单（逐项确认，防止漏填）

> 最高频使用文件。每次在后台填写文章信息时，先对照这里。

---

### [AI提示词工作流.md](./AI提示词工作流.md)
**AI 批量生产文章的完整流程，每次用 AI 写文章前用。**

包含 5 个步骤的完整提示词：

| 步骤 | 做什么 | 输出 |
|------|-------|------|
| Step 0 | 关键词挖掘 | 15~30 个真实搜索词（带搜索量级） |
| Step 1 | 痛点挖掘 | 该游戏 12~15 条专属痛点 |
| Step 2 | 角度矩阵生成 | N 篇文章任务卡（标题+核心词） |
| Step 3 | 每日任务筛选 | 今日最值得写的 5 篇（流量×转化双优先） |
| Step 4 | 文章正文生成 | 完整 Markdown 文章 |

还包含：游戏卡片库模板、人工验证关键词的工具和方法。

---

### [内容追踪矩阵.md](./内容追踪矩阵.md)
**进度管理文件，每发完一篇立刻更新。**

包含：
- 每款目标游戏的**素材卡**（Step 0~2 输出结果存档）
- 每款游戏已写/未写的角度（用 ✅ / ⬜ 标记）
- 全量**去重索引**（直接复制粘贴给 Step 3 用）
- 发布统计表

> 这是 AI工作流和发布记录之间的"中间数据库"，不维护它 Step 3 就无法避免重复选题。

## 三、完整工作流（从策略到发布）

```
战略层（月度对齐）
    SEO流量战略.md → 确认主站/子站分工、内链目标

        ↓

素材准备（每款新游戏初始化一次，约 20 分钟）
    AI提示词工作流.md Step 0 → 挖掘关键词（存入内容追踪矩阵）
    AI提示词工作流.md Step 1 → 提取痛点（存入内容追踪矩阵）
    AI提示词工作流.md Step 2 → 生成角度矩阵（存入内容追踪矩阵）

        ↓

每日生产（按需，约 30 分钟/篇）
    AI提示词工作流.md Step 3 → 选出今日 3~5 个任务（从内容追踪矩阵取去重索引）
    AI提示词工作流.md Step 4 → 逐篇生成文章

        ↓

发布前配置
    内容发布指南.md → 核对 SITE_ID、Section Slug、SEO字段、内链

        ↓

更新追踪
    内容追踪矩阵.md → 勾选已完成角度，更新去重索引
```

## 五、核心规则速记

```
主站（SITE_ID=1）：问答/截流/破解 → news-section
子站（SITE_ID=8）：截流/痛点/充值避坑 → news-section；攻略 → guide-section

每篇文章必须有：
  ① 游戏详情页内链 /games/[id]
  ② 主站文章链 /boxes/ 或 /01zhe/
  ③ 子站转化型链主站盒子页（跨站）
  ④ 推荐 3 篇相关文章（站内循环）

AI生产红线：
  ① 标题来自 Step 0 真实搜索词，不得拍脑袋造词
  ② 折扣数字一律写占位符 [折扣:X折待填]
  ③ 推荐游戏只能是子站已收录游戏
```

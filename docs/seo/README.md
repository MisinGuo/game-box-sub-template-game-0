## 目录结构

当前目录按用途拆成 6 层：

| 目录 | 放什么 |
|------|--------|
| `01-战略/` | 当前有效的总战略、主站与子站矩阵关系 |
| `02-规划/` | 候选子站登记、单站 7 仓位排期、示例排期 |
| `03-执行/` | AI 工作流、发布手册、追踪矩阵 |
| `04-示例/` | AI 文章样稿，给执行人员参考 |
| `05-验证/` | 技术 SEO 验收与检查 |
| `99-归档/` | 历史方案，只做追溯，不做当前执行标准 |

`materials/` 保留在根目录，是给执行流程逐步调用的材料库，不参与战略判断顺序。

`materials/` 现已继续按 Step 拆层：

| 子目录 | 作用 |
|------|------|
| `Step-0-决策前置/` | 不放 M 文件，只放执行前确认说明 |
| `Step-1-目标池与关键词/` | 选游戏、验词、确认词路 |
| `Step-2-痛点与机制/` | 提炼痛点、机制词和老玩家表达 |
| `Step-3-角度矩阵/` | 生成任务卡 |
| `Step-4-今日任务/` | 选出当天 5 篇任务 |
| `Step-5-正文发布与回写/` | 取真实卡片、写正文、发布、回写 |

进入 `materials/` 后，先看 [materials/README.md](./materials/README.md)，再按 Step 目录逐步往下走。

## 二、实施经理阅读顺序

如果你的工作是“按当前战略推动落地”，建议严格按下面顺序看文档：

1. [SEO流量战略.md](./01-战略/SEO流量战略.md)：先看项目层和站内层的双层赛马模型，不先看这个，后面所有材料都会被误读成单纯写稿流程。
2. [候选子站登记表.md](./02-规划/候选子站登记表.md)：确认当前有哪些站、每个站守什么边界、哪个站能继续投。
3. [单站7仓位排期模板.md](./02-规划/单站7仓位排期模板.md)：确认当前站的 A-G 仓位、阶段、周节奏和最近观察结果。
4. [AI提示词工作流.md](./03-执行/AI提示词工作流.md)：把 Step 0 到 Step 5 的执行口径交给员工，明确哪些是人工决策，哪些能交给 AI。
5. [内容发布指南.md](./03-执行/内容发布指南.md)：交代发布专员和审稿人员的后台操作标准。
6. [内容追踪矩阵.md](./03-执行/内容追踪矩阵.md)：看去重索引、完成状态、周/月复盘基础数据。
7. [SEO验证清单.md](./05-验证/SEO验证清单.md)：只有涉及技术改动或验收时再看，不属于每日内容流程起点。

实施经理不要这样看：

1. 不要先打开 `materials/Step-*` 再倒推战略。
2. 不要把 [主站+子站SEO架构调整-执行方案.md](./99-归档/主站+子站SEO架构调整-执行方案.md) 当成当前标准，它是历史归档。
3. 不要把 AI 工作流理解成“让模型自己决定选站、选池、选仓位”。

## 四、每个文件讲什么

### [SEO流量战略.md](./01-战略/SEO流量战略.md)
**整体战略文件，月看一次。**

回答"我们在做什么"：
- 主站（5awyx.com）vs 多候选子站矩阵的分工
- 整个站点的 URL 结构和页面类型
- 内链系统：每个页面指向谁、从哪里被指向
- Tag 页系统（流量放大器）

> 适合每月做一次战略对齐，或者有新人加入时介绍背景时看。

---

### [候选子站登记表.md](./02-规划/候选子站登记表.md)
**新开子站前先填的登记文件。**

回答“这个站能不能开、边界怎么定”：
- site_key、域名、负责人、关联模板来源
- 这个站只守哪个小类目，哪些词能做，哪些词不能抢
- 主站承接页绑到哪里，14~21 天和 30~60 天怎么判定胜负
- 当前有哪些候选站、赢家站、冻结站

> 适合每次准备新开一个候选子站时先建档。

---

### [单站7仓位排期模板.md](./02-规划/单站7仓位排期模板.md)
**单站 7 仓位起步排期表。**

回答“这个站这周到底怎么发”：
- A-G 仓位分别打哪组游戏 / 词路
- 每天 5 篇页面组怎么排
- 哪个仓位已经冒头，应该补 FAQ、对比页还是人工锚点页
- 60 天后这个站内哪些仓位保留、整站该不该扩量

> 适合每个候选子站单独复制一份，作为日常执行表。

---

### [内容发布指南.md](./03-执行/内容发布指南.md)
**发文操作手册，每次发文前查。**

回答"这篇文章该怎么填"：
- 后台 SITE_ID 和 Section Slug 怎么选（填错文章不展示）
- 主站 10+ 种文章类型的定义、标题模板、内链要求
- 子站 10+ 种文章类型的定义（截流/痛点/充值避坑等）
- 快速发文对照表（一张表覆盖所有情况）
- 发布前检查清单（逐项确认，防止漏填）

> 最高频使用文件。每次在后台填写文章信息时，先对照这里。

---

### [AI提示词工作流.md](./03-执行/AI提示词工作流.md)
**AI 批量生产文章的完整流程，每次用 AI 写文章前用。**

包含 6 个顺排步骤，并明确人工 / AI 分工：

| 步骤 | 做什么 | 输出 |
|------|-------|------|
| Step 0 | 先确认站点、仓位、阶段、池子 | 当天已确认执行边界 |
| Step 1 | 关键词挖掘 | 15~30 个真实搜索词（带搜索量级） |
| Step 2 | 痛点挖掘 | 该游戏 12~15 条专属痛点 |
| Step 3 | 角度矩阵生成 | N 篇文章任务卡（标题+核心词） |
| Step 4 | 每日任务筛选 | 今日最值得写的 5 篇（流量×转化双优先） |
| Step 5 | 文章正文生成 | 完整 Markdown 文章 |

还包含：员工每日执行清单、人工验证关键词的口径、AI 只能做建议不能做决策的边界。

---

### [内容追踪矩阵.md](./03-执行/内容追踪矩阵.md)
**进度管理文件，每发完一篇立刻更新。**

包含：
- 每款目标游戏的**素材卡**（Step 1~3 输出结果存档）
- 每款游戏已写/未写的角度（用 ✅ / ⬜ 标记）
- 全量**去重索引**（直接复制粘贴给 Step 4 用）
- 发布统计表

> 这是 AI工作流和发布记录之间的"中间数据库"，不维护它 Step 4 就无法避免重复选题。

## 五、实施顺序（从战略到日更）

### 阶段一：先定战略，不开写

1. 看 [SEO流量战略.md](./01-战略/SEO流量战略.md)，明确主站、子站矩阵、单站 7 仓位的关系。
2. 看 [候选子站登记表.md](./02-规划/候选子站登记表.md)，决定当前投哪个站，不投哪个站。
3. 看 [单站7仓位排期模板.md](./02-规划/单站7仓位排期模板.md)，决定今天跑哪个仓位。

### 阶段二：再把流程交给员工

1. 让执行员工按 [AI提示词工作流.md](./03-执行/AI提示词工作流.md) 从 Step 0 跑到 Step 5。
2. 让执行员工只在已确认 `site_key / slot / pool_type` 的边界内使用 `materials/`。
3. 让发布专员按 [内容发布指南.md](./03-执行/内容发布指南.md) 负责后台配置和上线检查。

### 阶段三：最后回写和复盘

1. 每篇发完都要更新 [内容追踪矩阵.md](./03-执行/内容追踪矩阵.md)。
2. 每周回看单站排期和追踪矩阵，决定是否加档、观察、冻结。
3. 涉及技术修正时，再切到 [SEO验证清单.md](./05-验证/SEO验证清单.md)。

## 八、核心规则速记

```
主站（SITE_ID=1）：问答/截流/破解 → news-section
当前候选子站（SITE_ID 以登记表为准）：截流/痛点/FAQ/充值避坑 → news-section；深度攻略/教程 → guide-section

发文前先查：02-规划/候选子站登记表.md → 确认当前 site_key / SITE_ID / 主题边界

每篇文章必须有：
  ① 游戏详情页内链 /games/[id]
  ② 主站文章链 /boxes/ 或 /01zhe/
  ③ 候选子站文章必须链主站盒子页（跨站）
  ④ 推荐 3 篇相关文章（站内循环）

AI生产红线：
  ① 标题来自 Step 0 真实搜索词，不得拍脑袋造词
  ② 折扣数字一律写占位符 [折扣:X折待填]
  ③ 推荐游戏只能是子站已收录游戏
```

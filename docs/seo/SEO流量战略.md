# 🔥 一、总战略（最终版）

> ✅ **SEO流量系统 + 转化系统 + 多站扩张系统**

------

## 👉 核心模型

```
子站内容（SEO流量）：攻略 + 新手教程 + 版本分析 + Meta分析
↓
文章内链游戏推荐 → 跳转主站游戏页
↓
主站流量承接：游戏页(/games/[id]) + 排行榜(/rank/) + Tag聚合(/tag/[tag]/)
↓
权重循环：0.1折专题(/01zhe/) + 盒子对比内容
↓
精准转化：游戏盒子(/boxes/) — 对比折扣、点击推广链接
```

> 💡 **核心分工原则**：内容和转化分工，不要混在一起。
> **主站 = 转化中心（产品站）**：承接流量 → 盒子下载/注册转化。
> **子站 = SEO内容中心（流量站）**：持续产出深度内容 → 导流主站。
> 最终实现：**流量 → 内容 → 信任 → 主站转化**。

## 🛡️ Google HCU（有用内容算法）风险防护

> HCU（Helpful Content Update）的核心判断：**这批内容是为人写的，还是为SEO写的？**
> 批量AI内容的最大风险是"结构高度相似 + 内容可替换"，被整站打标后排名全面下滑。
> 以下机制已内嵌在 [AI提示词工作流.md](./AI提示词工作流.md) 中，执行时必须严格遵守，不可跳过。

------

### 防护机制1：先验证搜索需求，再写文章（Step 0）

```
❌ 错误做法：批量生成标题，无论有没有人搜都发布
✅ 正确做法：每个标题的核心词必须经过 Step 0 关键词挖掘验证
           没有搜索量的关键词 = 不写
           工具：百度搜索框下拉联想 / 百度指数 / 5118 / 爱站
```

> 意义：只产出有真实需求的内容，让每篇文章都有潜在读者，从源头避免"为爬虫写内容"的信号。

------

### 防护机制2：游戏专属痛点，禁止套模板（Step 1）

```
❌ 低质量示例（AI套模板）："这款游戏太肝了，每天要做很多任务"
✅ 高质量示例（绑定真实游戏机制）："每天需完成12个日常任务，公会战强制在线，
   活动期间需连续签到7天" → 导致上班族/轻度玩家流失

判断标准：Step 1 输出的痛点描述中，
          如果没有出现该游戏独有的系统或机制名称，说明AI在套通用模板，该篇不可用。
```

对AI不熟悉的冷门游戏，必须使用「评论辅助版」：从 TapTap/App Store 差评区复制 10~20 条真实评论喂给 AI，提取真实痛点，而不是让 AI 自行发挥。

------

### 防护机制3：每篇核心论点必须不同（Step 4 质量红线）

Step 4 审核发布前，必须逐项通过以下检查：

| 检查项 | 说明 |
|---|---|
| 痛点是否具体 | 文章中的游戏问题描述必须有具体细节，不接受"太肝/太氪"等泛泛描述 |
| 推荐游戏是否子站有 | 不能推荐子站没有收录的游戏，防止死链 |
| 内链占位符是否替换 | 所有 `{{URL}}` 已替换为真实游戏/盒子URL |
| 折扣数据是否真实 | `[折扣:X折待填]` 已替换为主站后台实际价格，不接受AI编造 |
| 核心论点是否差异化 | 同一款游戏的不同角度文章，推荐理由必须不同，不能只换游戏名 |

------

### 防护机制4：发布节奏控制（先质量后放量）

```
阶段0（建站初期，第1~2周）：
  → 先发 30~50 篇经过完整 Step 0~4 流程的文章
  → 确认 Google 已索引且无异常降权信号后，再进入放量阶段

阶段1（放量起步，第3~4周）：
  → 10~20 篇/天
  → 每篇仍要过完整审核红线，不得走形式

阶段2（稳定放量）：
  → 30~80 篇/天（并发批次）
  → 同步保持人工精品文章每周 3~5 篇，作为E-E-A-T锚点
```

> ❗切忌在建站第一周就冲 80 篇/天。Google 对新站的初始评分窗口极为敏感，
> 大量结构相似内容会直接触发"站点级别"质量惩罚，后续很难恢复。

------

### 防护机制5：人工精品内容作为E-E-A-T锚点

```
AI 内容量再大，如果站上没有任何"真人写的、有专业深度的内容"，
Google 会将整站判定为 AI 内容农场。

人工精品文章（guide-section/review-section）的作用：
- 提供真实经验（Experience）和权威性（Authoritativeness）信号
- 让 Google 评估时有"高质量样本"参照
- 每周 3~5 篇，是 AI 批量内容能安全放量的前提条件，不可省略
```

## 16.1 为什么子站必须同时建「游戏类型页」+「攻略页」

| 维度 | 仅发攻略 | 游戏类型 + 攻略 |
|------|---------|----------------|
| 关键词覆盖 | 只覆盖"XX游戏攻略"类词 | 同时覆盖"仙侠游戏推荐"等大流量词 |
| 内容权威性 | 零散文章，缺乏结构 | 品类页作为 Hub，建立垂直领域权威 |
| 内链结构 | 各文章孤岛，权重难以集中 | 类型页↔攻略页双向内链，权重流动清晰 |
| 用户意图覆盖 | 只覆盖「如何玩」 | 同时覆盖「玩什么」+「如何玩」 |
| 对抗竞争 | 容易被单篇文章替代 | 品类权威站不易被单页超越 |

**结论：子站定位是「品类垂直站」，必须同时建设品类聚合页和攻略内容页，二者缺一不可。**

## 漏洞 #1 🔴 AI内容被 Google HCU/SpamBrain 识别风险

**状态：🔄 解决中**

**核心问题（重新定位）：**
> 问题不是 AI 占比多少，而是 **Google 能否识别出来** + **内容是否真正满足用户意图**。
> 这两个条件同时成立时才会被惩罚——识别出来但内容质量好的站不一定被罚，无法识别的 AI 内容不会被罚。

**Google HCU 实际惩罚的是：**
1. 内容与用户搜索意图不匹配（高跳出率、低停留）
2. 大量文章呈现可识别的模板化"AI指纹"（句式重复、过度补充说明、无原创信息）
3. 文章大量依赖泛化内容，无任何专属于本站的独特数据

**"破解截流型"的真正问题：**
- 用户搜索《XX破解版》→ 想知道有没有能用的破解版
- 文章内容却是《正规盒子才是正确选择》→ 意图完全错位
- 结果：用户打开 2 秒就跳出，Google 拿到的行为数据极差 → 比 AI 本身危害更大

### ✅ 解决方案

#### 方案A：破解截流型 → 安全顾虑型（内容角度重写）【已决策】

| 维度 | 破解截流型（旧）| 安全顾虑型（新）|
|------|--------------|----------------|
| 标题 | 《XX破解版是真的吗？正规盒子才是正确选择》 | 《XX游戏第三方平台安全吗？亲测3款盒子对比》 |
| 用户真实意图 | 想找破解版/省钱方式 | 担心封号/扣费/数据安全 |
| 内容方向 | 直接否定破解版 → 推盒子 | 分析风险 → 对比盒子安全性数据 → 推荐 |
| 行为信号 | 跳出率高（意图错位） | 停留时间长（真正回答了问题） |
| 内链 | /boxes/ | /boxes/[id] 具体安全说明 + 游戏页 |

**新标题规范示例：**
```
《XX游戏用第三方盒子会封号吗？实测数据说话》
《XX盒子靠谱吗？3个维度评测：资质/折扣/封号风险》
《手游福利服和官服有什么区别？哪个更安全》
```

#### 方案B：内容质量标准（防 HCU 核心检查规则）

> 每篇文章发布前必须过以下标准，AI 草稿不符合的必须修改后才能发布。

**一、内容有效性检查（1~5）**

**✅ 1. 这篇文章真的解决一个问题吗？**

发布前问自己3个问题：
- 用户为什么要搜这个关键词？
- 这篇文章有没有直接回答问题？
- 用户看完能不能"马上用"？

> 有一个"不确定" → 先别发

**✅ 2. 有没有"本站独有的东西"（最关键）**

必须至少包含一个：
- 本站游戏库数据（折扣率/评分/礼包）
- 真实使用对比（盒子A vs 盒子B 具体数据）
- 明确观点（不是总结别人，是我们的判断）

```
❌ 错误："根据研究…很多人认为…这款游戏不错"
✅ 正确："我们测试了XX盒子，首充折扣是0.1折，比官服省了75元"
```

**✅ 3. 是否避免拼凑内容**

检查：
- 有没有只是把多个网站内容合并？
- 有没有只是换句话说（洗稿）？

> 是 → 直接重写

**✅ 4. 内容深度够吗？**

快速标准：
- 是否比 Google 前3名更有用？
- 是否有：步骤 / 示例 / 对比 / 注意事项

> 只是"解释概念" → HCU 会直接干掉

**✅ 5. 有没有废话段落**

删掉这些：
- "在当今数字时代…"
- "越来越多的人开始…"
- 空洞总结、无信息量的开头

---

**二、结构优化（6~8）**

**✅ 6. 标题不过度 SEO**

```
❌ 错误："2026最佳XXX终极指南（超详细+免费）"
✅ 正确：直接说明解决什么问题，越自然越好
```

**✅ 7. 结构清晰**

必须做到：
- 有 H2 / H3 分段
- 一眼能扫读
- 步骤类内容有编号

**✅ 8. 不要关键词堆砌**

> 自然写 > SEO写，同一个词出现超过3次要检查

---

**三、发布前风险扫描（9~11，防 SpamBrain）**

**✅ 9. 不做黑帽外链操作**
- ❌ 不买外链 / 不群发外链 / 不用自动工具
- 新文章 0 外链没问题，等自然收录

**✅ 10. 内链要自然**
- 只做指向相关内容的内链
- ❌ 不强行插关键词锚文本

**✅ 11. 不隐藏/欺骗**
- ❌ 禁止隐藏文字、cloaking（不同用户看不同内容）

---

**四、站点层面（12~14）**

**✅ 12. 网站主题统一**
- 主站专注：游戏盒子折扣场景
- 子站专注：单一品类攻略
- ❌ 错误：一个站什么都写

**✅ 13. 控制内容质量比例**

> 关键指标：高质量内容 ≥ 70%

- 100篇文章里 70 篇是水文 → **整站都会被拖垮**（HCU 站点级惩罚机制）
- 每发 1 篇 AI 草稿水文，必须配 1 篇高质量内容"对冲"

**✅ 14. AI 使用规则**

```
✅ 可以：AI辅助写作 / AI整理结构 / AI加速初稿
❌ 不可以：直接生成不改就发 / 批量几十篇同一模板
```

---

**五、发布前最终5问（必须全 YES 才发布）**

```
1. 这篇文章是否真正解决一个问题？
2. 是否包含本站独有的数据/经验/观点？
3. 是否比现有排名内容更有价值？
4. 是否没有拼凑/洗稿？
5. 是否读起来像"人写的"？

→ 有一个 NO → 不发布
```

#### 方案C：AI指纹破除（避免被 SpamBrain 统计识别）

**问题根源：** 用同一套 Prompt 模板批量生产 → 文章结构/语气/句式高度相似 → 可被统计指纹识别

**执行规则：**

```
1. Prompt 多样化
   - 维护 3~5 套不同的 Prompt 模板（问答型/分析型/对比型/叙事型/建议型）
   - 同一类文章轮换使用，不连续使用同一模板
   - 禁止固定 Prompt 开头（如"作为一个游戏玩家..."）

2. 结构多样化
   - 文章格式轮换：Q&A格式 / 总结-展开 / 时间线 / 对比表格 / 数字列表
   - 正文长度变化：600字 / 900字 / 1200字 混排，不要全部相同长度

3. AI 禁用词/句型（写入 Prompt 约束）
   - "值得注意的是" / "此外" / "总的来说" / "综上所述"
   - "作为玩家" / "在游戏体验中" / "不得不说"
   - 固定格式总结段（每篇结尾要求不同）
```

#### 方案D：数据注入（给每篇文章制造唯一性锚点）

**原理：** 向 AI 内容中注入本站独有的真实数据，每篇文章有不可复制的数据锚点。

**数据来源（后端 API 已有）：**

```
- 游戏名称 + 当前折扣率（来自 ruoyi-gamebox 游戏库）
- 盒子平台对应折扣（来自 boxes API）
- 该游戏礼包信息（来自礼包中心）
```

**注入方式（Prompt 中动态插入）：**
```
"XX游戏在[盒子名]的当前折扣是[N折]，原价[X元]盒子价[Y元]"
"该游戏目前评分[N分]，收录于[品类]"
"当前可领礼包：[具体礼包名+URL]"
```

> 每篇文章 = AI泛化内容 + 实时真实数据 → 独特锚点无法被判为纯AI拼凑

#### 方案E：半自动写作流程 + 内容安全评分标准

> 这是把 AI 用对的核心：**人负责思考和经历，AI 负责补充和表达**

**一、标准文章骨架模板（适配本站游戏/盒子类内容）**

```
标题规范：【具体问题】+【解决结果】（有结果导向，不夸张，不标题党）

示例：
❌ "2026最佳游戏盒子终极指南（超详细+免费）"
✅ "XX游戏用哪个盒子最划算？我对比了3款，差价最高75元"
```

**开头（必须有"人味"）：**
```
我在玩XX游戏的时候，遇到了一个问题：XXX
试了很多方法（包括XXX），大部分没用。
后来发现了[盒子名]，实测折扣是[N折]，比官服省了[X元]。
这篇文章直接讲清楚怎么用。
```

**核心内容结构（每段必须带经验）：**
```
方法1：XXX
  我是怎么用的：（具体步骤）
  踩过的坑：（失败/问题）
  适合什么情况：（用户判断条件）

方法2：XXX（必须有对比）
  和方法1的区别：（直接对比，有数据）
  优点 / 缺点：
  我的实际建议：（给明确结论）
```

**反常识段落（加分项，必加）：**
```
很多人会以为：XXX
但实际情况是：XXX（本站数据支撑）
我的结论是：XXX
```

**结尾（不废话，直接给行动指引）：**
```
如果你只记住一件事：→ XXX（最关键结论）
新手：→ 建议从[方法X]开始
有基础：→ 直接用[方法X]
```

**二、AI 正确使用方式（三种安全用法）**

```
✅ 用法1：让 AI 写结构，你填内容
   Prompt："帮我整理一篇关于[XX盒子 vs 官服]的文章结构，包含方法/对比/常见错误"
   然后你加：本站游戏折扣数据 + 真实使用经验 + 你的观点

✅ 用法2：让 AI 改写你自己的内容
   Prompt："这是我写的一段内容，帮我优化表达，让它更清晰：[你的原文]"
   → 内容还是你的，AI只是优化表达

✅ 用法3：让 AI 补充细节
   Prompt："帮我补充这个步骤的细节：[你已有内容]"
   → 骨架是人写的，AI 填充说明

❌ 绝对不要这样用：
   "帮我写一篇XX文章" → 直接废掉（典型垃圾内容）
   批量生成几十篇同模板 → 整站足迹暴露
```

**三、半自动写作流程（比例是关键）**

```
Step 1：你写核心（占 20%）
  → 折扣数据 / 使用经验 / 你的观点 / 本站案例

Step 2：AI 扩展（占 50%）
  → 补结构 / 补说明 / 优化表达

Step 3：你再改（占 30%）
  → 加人话 / 加判断 / 删废话 / 加内链

原则：人的贡献 ≥ AI 的贡献
```

**四、内容安全评分标准（发布前自评）**

| 评级 | 特征 | 结果 |
|------|------|------|
| ✅ 安全 | 有个人/本站经验 + 有数据对比 + 有明确观点 + 结构清晰 + 不是拼凑 | 基本不会被 HCU 打 |
| ⚠️ 危险 | 大量AI生成 + 无本站数据 + 泛泛而谈 + 跟别人差不多 | 很容易掉排名 |
| ❌ 高危 | 批量生成+不改 / 洗稿 / 拼接 / 无任何价值 | 可能拖垮整站 |

> 核心原则：**AI 可以帮你写，但不能替你"思考和经历"**

#### 方案F：差异化 AI Prompt 模板库（5套轮换）

> 写入 AI 工作流配置，同一文章类型每次轮换使用不同模板，避免统计指纹。

**通用禁用词/句型约束（所有模板共用，放在每条 Prompt 末尾）：**

```
【写作禁止规则，必须遵守】
禁止使用以下词语和句型：
- "值得注意的是" / "此外" / "总的来说" / "综上所述" / "总而言之"
- "作为玩家" / "在游戏体验中" / "不得不说" / "毋庸置疑"
- "随着XX的发展" / "在当今时代" / "越来越多的人"
- "我们可以看到" / "不难发现" / "显而易见"
禁止以"总结"段落结尾，结尾必须是具体行动建议。
禁止使用固定开头句式，直接进入正题。
全文长度控制在[指定字数]左右，不要凑字数。
```

**模板1：问答型（Q&A格式，600~800字）**

```
你是一位真实玩过手游的玩家，用直接、实用的语气写作。
文章主题：[标题]
结构：第一段直接给答案 → Q1/A1 → Q2/A2 → Q3/A3 → 我的建议+链接
数据：游戏{{game_name}}，盒子{{box_name}}折扣{{discount}}，省{{save_amount}}元
[禁用词规则]
```

```
[禁用词规则]
```

**模板3：叙事型（经历故事，700~900字）**

```
结构：具体场景开头 → 遇到问题 → 发现盒子 → 用后对比（有数字）→ 建议 → 行动链接

[禁用词规则]
```

**模板4：步骤教程型（数字步骤，800~1000字）**

```
数据：盒子{{box_name}}，游戏{{game_name}}，折扣{{discount}}，礼包{{gift_name}}
```

```
结构：说明评选标准 → 排行主体（每条：名称+核心数据+推荐理由+适合人群）→ 怎么选的判断逻辑+链接
```

## 漏洞 #2 ✅ 内链枢纽页缺失问题已解决

**状态：✅ 已解决（2026-04-18）**

**原问题描述：**
- 战略核心内链路径：`/news/[id] → /games/[id] → /rank/ → /01zhe/ → 权重循环`
- `/01zhe/`（0.1折专题）、`/rank/`（排行榜）、`/tag/[tag]/`（Tag聚合页）三个节点**全部未建设**
- 已发布的文章内链如果指向这三个页面，实际上是死链

## 漏洞 #4 ✅ 子站转化路径优化（已采用替代方案）

**状态：✅ 已解决（2026-04-18）**

**原问题描述：**
- 子站当前转化路径：子站读者 → 点击CTA → 跳转主站品类页 → 找盒子 → 点推广链接 → 注册
- 这是 4 步漏斗，每增加一步流失 40~60%

**已实施的优化：**
- 子站 DownloadBox 和 ArticleLayout 的 CTA 按钮现在**直接链接到主站 `/boxes` 页面**
- 缩短为 3 步漏斗：`子站文章 → 主站/boxes → 点推广链接 → 注册`

**为何不链接到游戏页：**
- 链接到游戏页需要推荐系统决定推荐哪个游戏
- 链接到 `/boxes` 页面让用户可以自由选择或搜索想玩的游戏，体验更好
- 所有外链已添加 `rel="nofollow noopener noreferrer"`，符合链接规范

# 候选子站登记表

> 用途：在任何候选子站开始发文前，先把站点边界、导流目标、淘汰标准登记清楚。
> 维护规则：一站一张登记卡；状态变化时只追加记录，不覆盖旧判断。
> 结构口径：A-G 7 仓位只用于单站内部排期、赛马观察和资源倾斜，不作为公开 sitemap 或栏目结构。

---

## 一、怎么用

### 新开一个候选子站前

1. 先确认这个站只守 1 个小类目，不和现有子站抢词。
2. 复制第二节的“子站登记卡模板”，粘贴到第三节“候选子站卡库”。
3. 同时复制 [单站7仓位排期模板.md](./单站7仓位排期模板.md) 给这个站单独排期。
4. 没建登记卡，不开站，不发文，不进 AI 工作流。

### 运行中怎么维护

1. 14~21 天补一次“最近判断”，先看站内 7 仓位有没有人冒头。
2. 30~60 天再补一次“最近判断”，决定继续扩量、冻结还是淘汰。
3. 如果站点边界、导流目标、域名、负责人有变化，直接更新登记卡顶部字段。
4. 公开页面结构继续按站点原有板块 / 分类维护；仓位变化只更新排期和后台观察口径。

## 五、状态口径速记

```text
Candidate：刚起步，先看站内 7 仓位有没有人冒头
Winner：当前小类目已起量，准备扩更大词、更多游戏和更深内容
Loser：站内长期无人冒头，整站价值偏低，准备冻结或淘汰
Frozen：停更观察，只保留存量页面和历史数据
```

---

## 六、硬规则

```text
1. 一个候选子站前期只守 1 个小类目，不允许边跑边扩成大杂烩。
2. 每个站都必须绑定主站承接页，首批页面就开始导流。
3. 没有“最近判断”的站，不能直接进入下一轮扩量。
4. 项目总量 = 活跃子站数 × 单站节奏；资源不够时减少活跃站，不要半死不活地全开。
```

# pSEO 子站游戏池筛选方法论（三源合流优化版）

> **核心原则：** 榜单定周期，词根定意图。用大盘老游做基本盘，用新游抢占红利，用下沉市场（安卓/小游戏）做精准转化。
> **适用阶段：** 新子站启动前，确定7仓位主打游戏和扩展游戏池时使用。

---

## 一、为什么不能从盒子库存倒推

| 从盒子倒推（错误） | 从搜索需求出发（正确） |
|---|---|
| 盒子里有什么游戏 → 做什么站 | 什么词有搜索量 → 做什么站 |
| 只推荐盒子里的游戏 | 用热门游戏做钩子，推荐盒子里有的同类 |
| 子站 = 盒子游戏的攻略站 | 子站 = 搜索流量的入口站 |
| 游戏池受限于盒子库存 | 游戏池 = 整个市场的搜索需求 |
| 小游戏搜索量低，做了也没流量 | 大游戏搜索量高，截流后导盒子转化 |

**一句话：盒子库存决定的是"能转化什么"，不是"能写什么"。**

---

## 二、三源合流漏斗模型

```
阶段1：三源合流全景扫描（从榜单出发）
    ↓ 不再爬取发布时间，榜单类型自带生命周期标签
阶段2：三轨制分流清洗（分流处理）
    ├─ 新游绿道：免检直通
    ├─ 高转平替：快速痛点确认
    └─ 大盘老游：三层漏斗深度筛选（AI粗筛→搜索验证→内容可行性）
阶段3：三轨汇合·词根碰撞（漏斗核心）
    ↓ 词根意图判定 → 淘汰无需求或竞争过高的游戏
阶段4：7仓位精准落位与实战策略
    ↓ 根据词根意图和来源标签，将优质游戏装填进7个子站仓位
阶段5：最终执行检查清单
```

---

### 阶段 1：三源合流全景扫描（突破时间与平台限制）

**目标：** 不再费时爬取具体发布时间，直接在抓取源头通过榜单类型赋予游戏生命周期标签。放弃纯 iOS 视角，全面拥抱高转化生态。

#### 1.1 建立三大底层候选池

| 数据源及榜单 | 隐含生命周期/受众标签 | 后续核心用途 |
|---|---|---|
| TapTap 预约榜 + 七麦 iOS/安卓飙升榜 | 爆发期新游（无历史数据但热度极高）**[新游绿道]** | 抢占初期长尾空白，吃流量红利 |
| 七麦 iOS 畅销榜 Top 100 | 稳定期老游大作（玩家基数大）**[大盘老游]** | 挖掘长尾痛点词，做深度攻略矩阵 |
| 微信小游戏畅销榜 + 华为/小米畅销榜 | 下沉市场爆款（极度契合盒子0.1折生态）**[高转平替]** | 作为极致诱饵，截流洗库直达转化 |

#### 1.2 产出原始表

将上述三张表合并去重，得到一张带有物理标签的**《多端游戏候选总表》**。每条记录自带来源标签（新游绿道/大盘老游/高转平替），后续阶段按标签分流处理。

---

### 阶段 2：三轨制分流清洗（解决大模型幻觉与新游漏筛）

**目标：** 拿到带有标签的《候选总表》后，按标签进行分流处理，扬长避短。

> **核心问题：** 大模型（AI）对刚上线的新游一无所知，如果统一丢给 AI 筛选，新游会被误杀；而老游和下沉游戏则可以通过 AI 高效剔除水分。同时，大盘老游和高转平替的筛选逻辑差异很大——老游需要深度验证搜索需求和内容可行性，平替只需快速确认"太氪"痛点是否存在。因此必须"三轨分流"。

#### 2.1 第一轨：【新游绿道】免检直通车

**动作：** 凡是带有 **[新游绿道]** 标签的游戏，**绝对不要送给大模型筛选**，直接进入阶段3的搜索量验证。

**目的：** 百分之百保全大模型知识库外的新兴爆款。新游正处于搜索长尾空白期，全网无竞品内容，是天然的流量红利窗口，错过即损失。

#### 2.2 第二轨：【高转平替】快速痛点确认

**动作：** 对带有 **[高转平替]** 标签的游戏（微信小游戏畅销榜 + 华为/小米畅销榜），进行轻量级筛选。

**筛选逻辑：** 这类游戏的核心价值是"太氪→0.1折"的转化路径，不需要深度验证内容可行性，只需确认痛点存在：

1. **AI 快速过滤：** 用大模型剔除纯休闲/无养成深度的游戏（同旧版 Prompt，但范围更小、更精准）
2. **痛点确认：** 搜"游戏名+太氪/充值/0.1折"，只要下拉框出现这类词根，直接通过
3. **盒子库存匹配：** 确认盒子里有该游戏的安卓换皮版或同类0.1折版

**提效结果：** 高转平替池通常只有 20~30 个游戏，筛选极快，10 分钟内可完成。

#### 2.3 第三轨：【大盘老游】三层漏斗深度筛选

**动作：** 对带有 **[大盘老游]** 标签的游戏（七麦 iOS 畅销榜 Top 100），执行完整的三层漏斗筛选。

> **为什么大盘老游需要完整漏斗？** 畅销榜 Top 100 里鱼龙混杂——有原神、王者荣耀这种搜索量极高但竞争也极高的 C 级词，有阴阳师这种衰退期红利型 S 级词，也有大量消消乐、音游等根本写不出攻略的水分游戏。不能简单丢给 AI 一句话过滤，必须逐层验证。

##### 第一层：AI 前置粗筛（100→30）

**目标：** 用大模型快速剔除无内容深度的游戏，大幅减少后续验证基数。

**Prompt 示例：**

```
我这里有一份 iOS 畅销榜 Top 100 的手游名单（附后）。请你扮演资深游戏运营，帮我筛选出满足以下条件的硬核/中重度游戏，剔除单机、超休闲和纯换皮无养成深度的游戏：
1. 游戏内有复杂的卡牌、角色或阵容搭配机制
2. 有明显的数值养成坑点（太肝/太氪）
请以表格形式返回符合条件的游戏名，并简述其核心玩法。
```

**提效结果：** AI 瞬间把 100 个游戏缩减到 30 个左右真正值得做 pSEO 的"有内容深度"的游戏。

##### 第二层：搜索需求验证（30→15）

**目标：** 对粗筛后的 30 个游戏批量验证搜索需求，筛掉没有搜索量或竞争过高的词路。

**（1）批量自动化验证——脚本替你完成体力活**

最一劳永逸的方法是写一个轻量级 Python 脚本，直接批量调用公开的下拉词接口：

```
http://suggestqueries.google.com/complete/search?client=chrome&q=游戏名+意图后缀
```

**操作思路：** 将游戏名列表存入 CSV，脚本循环请求该接口（加上"攻略"、"阵容"、"零氪"等意图后缀）。如果返回结果包含意图词，记为 True；返回全不相关或为空，直接淘汰。

**（2）批量拉取指数（可选 API 接驳）**

| 市场 | 工具 | 说明 |
|---|---|---|
| 国内 | 5118 API | 批量传入游戏名，返回百度指数和长尾词数量 |
| 海外/Google | DataForSEO | 按次付费 API，批量查询 Search Volume |

**（3）竞品逆向反推（交叉验证）**

最精准的搜索需求已被同行验证过。用 Ahrefs/5118 导出 3~5 个竞品站的关键词，提取词频最高的游戏名，直接锁定天然主打游戏。

**（4）游戏分级（按搜索量和竞争度）**

| 级别 | 特征 | 在子站中的角色 | 示例 |
|---|---|---|---|
| **S级：衰退期大游** | 曾经热门、搜索量仍有但竞争已退潮 | 截流型文章的钩子游戏 | 率土之滨、阴阳师、Fate/GO |
| **A级：中热度同类** | 搜索量稳定、同品类竞争适中 | 仓位主打游戏 | 少年三国志2、神仙道3 |
| **B级：盒子里的小游戏** | 搜索量低但盒子有折扣 | 截流文章中的"被推荐对象" | 盒子里的0.1折三国卡牌 |
| **C级：大热门主词** | 搜索量极高但竞争激烈 | 只做长尾词，不做主词 | 原神、王者荣耀 |

**淘汰标准：** 如果脚本跑不出任何长尾词根，或者词根全被九游、17173 等高权重老站垄断，直接淘汰。

##### 第三层：内容可行性判断（15→7~10）

**目标：** 确认每款候选游戏能支撑 20+ 篇不重复内容，筛掉写几篇就断的游戏。

**（1）内容深度检查**

| 检查项 | 操作 | 通过标准 |
|---|---|---|
| 痛点数量 | 搜"XX游戏+坑/氪/肝/退坑"看社区讨论 | ≥5 条具体痛点 |
| 版本更新频率 | 查游戏官网/公告更新记录 | 至少月更 = 有持续内容素材 |
| 阵容/角色数量 | 查游戏 wiki/攻略站 | ≥30 角色 = 阵容搭配空间大 |
| 社区活跃度 | 查 TapTap、贴吧、NGA 讨论量 | 有活跃讨论 = 有真实痛点可挖 |
| 多版本可写内容 | 查是否有专服/折扣版/怀旧版 | 多版本 = 多角度内容（版本对比、哪个版本适合XX） |

**（2）一票否决原则**

在进入打分前，先用这两个条件无情砍掉一批游戏：

- **内容枯竭否决：** 游戏角色/门派/职业少于 15 个，或已停更超过半年。写不到一个月就会"词穷"，直接淘汰。
- **零转化否决：** 该游戏既不在盒子里，盒子里也没有同类平替。完全没有承接路径，流量再大也是废流，直接淘汰。

**（3）内容角度穷举**

一款游戏如果能覆盖以下 5 类以上角度，就值得作为仓位主打：

1. **新手入门攻略**（必做）
2. **阵容/卡组推荐**（必做）
3. **零氪/低氪攻略**（必做）
4. **版本/Meta 分析**（有版本更新就做）
5. **充值避坑/折扣对比**（有折扣就做）
6. **FAQ 常见问题**（必做）
7. **截流型**（用大游戏名做钩子，推荐盒子里有的同类）
8. **痛点解决型**（太肝/太氪/怕停服）

**不值得做仓位主打的游戏特征：** 搜索量极低、没有版本更新、角色少于 10 个、社区几乎无讨论、纯工具类/休闲类。

##### 第三层产出：大盘老游精选池

经过三层漏斗后，大盘老游池从 100 个缩减到 7~10 个经过充分验证的优质游戏，每个都：
- 有真实搜索需求且竞争可切入
- 能支撑 20+ 篇不重复内容
- 至少有一条转化承接路径（直接或间接）

### 阶段 3：三轨汇合·词根碰撞（漏斗核心）

**目标：** 将三轨清洗后的名单汇合，通过自动化脚本请求 Google/百度下拉框及相关搜索，提取高频词根定性，为阶段4的仓位落位提供意图依据。

> **核心逻辑：** 不只看"有没有搜索量"，更要看"搜的是什么"。词根决定了用户的真实意图，意图决定了内容策略和转化路径。
>
> **三轨汇合说明：** 新游绿道免检直通到此，高转平替快速确认痛点后到此，大盘老游经三层漏斗精选后到此。三轨游戏在此统一接受词根碰撞验证，但验证侧重点不同：
> - 新游绿道：验证"兑换码/节奏榜/什么时候出"等新游意图词根是否存在
> - 高转平替：验证"太氪/0.1折/私服"等痛点词根是否存在
> - 大盘老游：已在第二层搜索需求验证中跑过一轮，此处重点确认词根意图类型（深度攻略 vs 极致截流）

#### 3.1 词根意图判定矩阵

批量提取到的高频词根，按以下矩阵判定游戏所处生命周期和对应内容策略：

| 词根类型 | 游戏实际状态 | 判定 | 内容策略与承接路径 | 转化潜力 |
|---|---|---|---|---|
| 下载、什么时候出、兑换码、节奏榜（新游） | 处于公测前/爆发期 | **新游红利** | 铺设"开荒避坑、角色节奏榜"。优先推荐盒子内同类高折扣游戏进行提前截流 | 高 |
| 最强阵容、太氪了、卡关、零氪（老游） | 处于稳定运营期 | **深度攻略** | 铺设"零氪/微氪生存指南"。做深度攻略，链接盒子转化 | 中→高 |
| 怀旧服、私服、停服、0.1折、平替（老游） | 处于衰退/核心玩家流失期 | **极致截流** | 铺设"平替推荐、官服vs折扣服对比"。极致截流，直导盒子 | 极高 |

#### 3.2 淘汰标准

如果通过脚本跑不出任何上述长尾词根，或者词根全被九游、17173 等高权重老站垄断，直接在这一步淘汰。

#### 3.3 批量验证脚本（实操参考）

**Google 下拉词批量请求：**

```
http://suggestqueries.google.com/complete/search?client=chrome&q=游戏名+意图后缀
```

将候选名单存入 CSV，写 Python 脚本循环请求，加上"攻略"、"阵容"、"零氪"等意图后缀。如果返回结果包含设定的意图词，记为 True；返回全不相关或为空，直接淘汰。

**批量拉取指数（可选 API 接驳）：**

| 市场 | 工具 | 说明 |
|---|---|---|
| 国内 | 5118 API | 批量传入游戏名，返回百度指数和长尾词数量 |
| 海外/Google | DataForSEO | 按次付费 API，批量查询 Search Volume |

**竞品逆向反推（交叉验证）：** 用 Ahrefs/5118 导出 3~5 个竞品站的关键词，提取词频最高的游戏名，直接锁定天然主打游戏。

### 阶段 4：7仓位精准落位与实战策略

**目标：** 根据阶段3跑出的词根意图和来源标签，将留存的优质游戏装填进7个子站仓位。

> **核心变化：** 不再用"三维打分+定桩法"从总分排名选游戏，而是直接按**来源标签+词根意图**匹配仓位类型。每个仓位类型有明确的选品来源和实战策略，落地性更强。

#### 4.1 三类仓位定义与选品规则

| 仓位数量 | 战略定位 | 选品来源与条件 | 实战策略 | 示例 |
|---|---|---|---|---|
| 2~3个 | **截流红利仓** | 必须来自 **[新游绿道]**。下拉框出现"兑换码/节奏榜" | 借新游全网无攻略的空档期狂铺内容。由于盒子大概率没这款新游，文章末尾顺势抛出："等公测太久/福利太少？不如先玩这款画风一致的0.1折版" | TapTap预约榜新游 → 推荐盒子内同类0.1折 |
| 2~3个 | **平替转化仓** | 必须来自 **[高转平替]**（微信小游戏尤佳）。下拉框出现"太氪/私服" | 针对《寻道大千》《咸鱼之王》等重氪小游戏，直接写痛点解决文，转化至盒子里数值相似、福利拉满的安卓换皮版 | 咸鱼之王太氪 → 盒子内0.1折同类 |
| 1~2个 | **直转攻略仓** | 来自 **[大盘老游]**，且盒子内有完全匹配的游戏库存 | 老老实实做深度阵容攻略，打造长尾矩阵，通过优质内容建立信任后实现直接转化 | 少年三国志2攻略 → 盒子折扣页 |

#### 4.2 仓位落位示例——三国卡牌子站

| 仓位 | 游戏 | 来源标签 | 词根意图 | 盒子有 | 内容策略 | 承接方式 |
|---|---|---|---|---|---|---|
| 1 | 某新游（TapTap预约Top3） | 新游绿道 | 兑换码/节奏榜 | ❌ | 截流型：开荒指南 → "等公测？先玩这款0.1折" | 间接承接 |
| 2 | 某新游2（七麦飙升榜） | 新游绿道 | 什么时候出/下载 | ❌ | 截流型：角色节奏榜 → 推荐同类0.1折 | 间接承接 |
| 3 | 寻道大千 | 高转平替 | 太氪/私服 | ✅（安卓换皮版） | 痛点型："太氪？0.1折版福利拉满" | 直接承接 |
| 4 | 咸鱼之王 | 高转平替 | 太氪/0.1折 | ✅（安卓换皮版） | 痛点型：官服vs盒子服对比 | 直接承接 |
| 5 | 少年三国志2 | 大盘老游 | 最强阵容/零氪 | ✅ | 攻略型：阵容/零氪/版本深度攻略 | 直接承接 |
| 6 | 神仙道3 | 大盘老游 | 阵容推荐/FAQ | ✅ | 攻略型：阵容/零氪/FAQ | 直接承接 |
| 7 | 阴阳师 | 大盘老游 | 怀旧/平替 | ❌ | 截流型：阴阳师攻略 → 推荐二次元0.1折 | 间接承接 |

#### 4.3 仓位间不互相竞争检查

7个仓位之间不能出现：
- 同一个搜索词被两个仓位同时做
- 两个仓位的文章标题高度相似
- 同一款游戏占据多个仓位（除非是不同专服版本，且搜索意图不同）

#### 4.4 确认扩展游戏池

7仓位主打游戏之外，再确认一批扩展游戏池，用于：
- 截流文章中的"被推荐对象"
- 品类聚合导航文章的收录范围
- 后续仓位轮换的候选

扩展游戏池来源：
1. 盒子里有的同类游戏（优先，有直接转化路径）
2. 搜索量中等但竞争低的同类游戏
3. 新游（公测前1~2周截流）

| 阶段 | 检查项 | 示例 | 完成 |

## 五、常见错误

| 错误 | 正确做法 |
|---|---|
| 从盒子库存倒推选题 | 从搜索需求出发选题 |
| 只做盒子里有的游戏 | 用热门游戏做钩子，盒子里有的做转化 |
| 7仓位全是盒子小游戏 | 新游截流 + 小游戏平替转化 + 老游深度攻略 |
| 认为盒子没有就不能写 | 截流型文章的核心就是写盒子没有的热门游戏 |
| 一上来就做"大品类词" | 先从具体游戏名+意图词切入，再做品类聚合 |
| 选了搜索量高但竞争全是大站的词 | 优先选竞争退潮的衰退期大游 |
| 7仓位选了7款同类同质游戏 | 三类仓位覆盖不同来源和意图（新游/平替/老游） |
| 逐一手工查搜索需求 | 用 AI 粗筛 + 脚本批量验证 + 竞品逆向反推 |
| 把新游也丢给大模型筛选 | 新游免检直通，只对老游/平替用 AI 提纯 |
| 大盘老游直接丢给AI一句话过滤 | 大盘老游需经三层漏斗（AI粗筛→搜索验证→内容可行性）逐步过滤 |
| 大盘老游和高转平替用同一套筛选逻辑 | 高转平替只需确认痛点存在，大盘老游需深度验证搜索需求和内容可行性 |
| 只看 iOS 畅销榜 | 三源合流：iOS + 安卓 + 微信小游戏 |
| 选了角色少、停更的游戏做仓位 | 用一票否决剔除内容枯竭和零转化游戏 |
| 仓位落位凭感觉 | 按来源标签+词根意图匹配仓位类型 |

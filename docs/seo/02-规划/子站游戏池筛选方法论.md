# 子站游戏池筛选方法论

> **核心原则：** 先找搜索需求，再找承接方式，最后才看盒子有没有。游戏是钩子，用户是目标，盒子是转化路径。
> **适用阶段：** 新子站启动前，确定7仓位主打游戏和扩展游戏池时使用。

---

## 一、为什么不能从盒子库存倒推

| 从盒子倒推（错误） | 从搜索需求出发（正确） |
|---|---|
| 盒子里有什么游戏 → 做什么站 | 什么词有搜索量 → 做什么站 |
| 只推荐盒子里的游戏 | 用热门游戏做钩子，推荐盒子里有的同类 |
| 子站 = 盒子游戏的攻略站 | 子站 = 搜索流量的入口站 |
| 游戏池受限于盒子库存 | 游戏池 = 整个市场的搜索需求 |
| 小游戏搜索量低，做了也没流量 | 大游戏搜索量高，截流后导盒子转化 |

**一句话：盒子库存决定的是"能转化什么"，不是"能写什么"。**

## 二、三层漏斗模型

```
第一层：搜索需求验证（从市场出发）
    ↓ 筛掉没搜索量、竞争过高的词路
第二层：内容可行性判断（从内容出发）
    ↓ 筛掉写不出20篇不重复内容的游戏
第三层：转化承接确认（从盒子出发）
    ↓ 确认至少有一条转化路径，但不是硬性前提
```

### 第一层：搜索需求验证

**目标：** 确认这个游戏/品类有真实搜索需求，且竞争可切入。

#### 2.1 搜索量验证

| 验证方式 | 操作 | 判断标准 |
|---|---|---|
| Google 搜索下拉 | 输入"XX游戏"看自动补全 | 有3条以上补全 = 有搜索需求 |
| 相关搜索 | 搜索结果页底部"相关搜索" | 有攻略/怎么玩/阵容类相关词 = 需求真实 |
| Google Trends | 对比几个候选游戏的搜索趋势 | 趋势稳定或上升 = 值得做 |
| 百度指数 | 查游戏名的搜索指数 | 日均搜索 > 500 = 有基础量 |
| 竞品站分析 | 搜目标词看前3页是什么站 | 全是大站占满 = 竞争过高，换词路 |

#### 2.2 游戏分级（按搜索量和竞争度）

| 级别 | 特征 | 在子站中的角色 | 示例 |
|---|---|---|---|
| **S级：衰退期大游** | 曾经热门、搜索量仍有但竞争已退潮 | 截流型文章的钩子游戏 | 率土之滨、阴阳师、Fate/GO |
| **A级：中热度同类** | 搜索量稳定、同品类竞争适中 | 仓位主打游戏 | 少年三国志2、神仙道3 |
| **B级：盒子里的小游戏** | 搜索量低但盒子有折扣 | 截流文章中的"被推荐对象" | 盒子里的0.1折三国卡牌 |
| **C级：大热门主词** | 搜索量极高但竞争激烈 | 只做长尾词，不做主词 | 原神、王者荣耀 |

#### 2.3 搜索意图分类

| 意图类型 | 搜索词示例 | 子站内容类型 | 转化潜力 |
|---|---|---|---|
| 攻略型 | "XX阵容推荐""XX怎么玩" | 攻略页（guide-section） | 中→高 |
| 痛点型 | "XX太氪了""XX零氪能玩吗" | 痛点解决页（guide-section） | 高 |
| 对比型 | "XX vs YY""XX哪个版本好" | FAQ/对比页（guide-section） | 高 |
| 截流型 | "XX同类游戏""XX平替" | 截流推荐页（news-section） | 高 |
| 下载型 | "XX下载""XX官网" | ❌ 不做，归主站 | — |
| 品牌型 | "游戏盒子""0.1折手游" | ❌ 不做，归主站 | — |

### 第二层：内容可行性判断

**目标：** 确认这款游戏能支撑20+篇不重复内容，不会写几篇就断。

#### 2.4 内容深度检查清单

| 检查项 | 操作 | 通过标准 |
|---|---|---|
| 痛点数量 | 搜"XX游戏+坑/氪/肝/退坑"看社区讨论 | 能列出5条以上具体痛点 |
| 版本更新频率 | 查游戏官网/公告更新记录 | 至少月更 = 有持续内容素材 |
| 阵容/角色数量 | 查游戏wiki/攻略站 | 30+角色/武将 = 阵容搭配空间大 |
| 社区活跃度 | 查贴吧/NGA/TapTap讨论量 | 有活跃讨论 = 有真实痛点可挖 |
| 专服/版本数量 | 查是否有0.1折/免费版/怀旧版等 | 多版本 = 多角度内容（版本对比、哪个版本适合XX） |

#### 2.5 内容角度穷举

一款游戏如果能覆盖以下5类以上角度，就值得作为仓位主打：

```
1. 新手入门攻略（必做）
2. 阵容/卡组推荐（必做）
3. 零氪/低氪攻略（必做）
4. 版本/Meta分析（有版本更新就做）
5. 充值避坑/折扣对比（有折扣就做）
6. FAQ常见问题（必做）
7. 截流型（用大游戏名做钩子，推荐盒子里有的同类）
8. 痛点解决型（太肝/太氪/怕停服）
```

#### 2.6 不值得做仓位主打的游戏特征

- 搜索量极低（搜不到相关词）
- 没有版本更新（写完一轮就没新素材）
- 角色少于10个（阵容搭配写不出几篇）
- 社区几乎无讨论（痛点都是编的）
- 纯工具类/休闲类（没有养成深度，写不出攻略）

### 第三层：转化承接确认

**目标：** 确认至少有一条转化路径，但不是硬性前提。

#### 2.7 三级承接关系

| 承接级别 | 条件 | 转化路径 | 内容策略 |
|---|---|---|---|
| **直接承接** | 盒子里有这款游戏 | 文章 → 子站游戏页 → 主站盒子页 → 下载 | 攻略型，直接推荐盒子折扣 |
| **间接承接** | 盒子里没有这款游戏，但有同类游戏 | 文章（热门游戏做钩子）→ 推荐盒子里有的同类 → 主站盒子页 → 下载 | 截流型，"喜欢XX？这几款同类0.1折" |
| **品牌承接** | 盒子里连同类都没有 | 文章 → 主站品牌页/盒子列表页 → 用户自选 | 品类聚合型，导主站整体 |

> **关键：** 直接承接最理想，但间接承接才是子站的主要模式。截流型文章的核心逻辑就是用大游戏的搜索量做钩子，把用户导向盒子里有的同类游戏。

#### 2.8 盒子库存的正确用法

盒子库存不是"能写什么"的限制，而是"转化时推荐什么"的素材库：

```
错误：盒子里有69款传奇游戏 → 所以做传奇站
正确：传奇品类搜索量大 → 所以做传奇站 → 文章里推荐盒子里有的传奇游戏

错误：盒子里没有热血传奇 → 所以不能写热血传奇
正确：热血传奇搜索量大 → 写热血传奇截流文章 → 推荐盒子里有的传奇0.1折手游

错误：7仓位的主打游戏必须都在盒子里
正确：7仓位 = 搜索需求最大的7条词路 → 盒子有的直接转化，没有的截流转化
```

## 三、子站游戏池筛选执行流程（详细版，含示例）

---

### 阶段 0：准备阶段

#### 明确目标

- **启动新子站**：确定子站所属小类目（如三国卡牌、传奇类、二次元等）
- **内容方向**：攻略 / 截流 / 推荐（三种方向可混合，但需明确主次）
- **转化路径**：文章 → 子站游戏页 → 主站盒子页 → 下载

#### 准备工具

| 类别 | 工具 | 用途 |
|---|---|---|
| 搜索工具 | Google、百度 | 验证搜索需求、发现长尾词 |
| 数据工具 | 七麦数据、Google Trends、百度指数、5118.com | 量化搜索量、对比趋势 |
| 社区/热度验证 | Tap、贴吧、NGA、B站 | 验证社区活跃度、挖掘痛点 |

> **建议：** 用 Excel 或 Google Sheet 建一张总表，每个阶段对应一列，方便填数据和打勾。七麦数据导出后直接填游戏名、热度、下载量 → 阶段1~2自动生成分级和初步备注 → 阶段3~4补充内容角度和承接方式 → 阶段5分配仓位。

### 阶段 4：转化承接确认

**目标：** 确认每款仓位主打游戏至少有一条转化路径，但不是硬性前提。

#### 4.1 三级承接关系

| 承接级别 | 条件 | 转化路径 | 内容策略 |
|---|---|---|---|
| **直接承接** | 盒子里有这款游戏 | 文章 → 子站游戏页 → 主站盒子页 → 下载 | 攻略型，直接推荐盒子折扣 |
| **间接承接** | 盒子里没有，但有同类游戏 | 文章（大游戏做钩子）→ 推荐盒子里有的同类 → 主站盒子页 → 下载 | 截流型，"喜欢XX？这几款同类0.1折" |
| **品牌承接** | 盒子里连同类都没有 | 文章 → 主站品牌页/盒子列表页 → 用户自选 | 品类聚合型，导主站整体 |

> **关键：** 直接承接最理想，但间接承接才是子站的主要模式。截流型文章的核心逻辑就是用大游戏的搜索量做钩子，把用户导向盒子里有的同类游戏。

#### 4.2 承接方式示例

| 游戏名 | 盒子是否有 | 承接级别 | 钩子→推荐路径 | 内容策略 |
|---|---|---|---|---|
| 阴阳师 | ❌ 无 | 间接承接 | 阴阳师 → 推荐"仙灵外传"等二次元0.1折 | 截流型 |
| 少年三国志2 | ✅ 有 | 直接承接 | 少年三国志2攻略 → 盒子折扣页 | 攻略型 |
| 率土之滨 | ❌ 无 | 间接承接 | 率土之滨 → 推荐"三国志战棋版"等SLG折扣 | 截流型 |
| 神仙道3 | ✅ 有 | 直接承接 | 神仙道3攻略 → 盒子折扣页 | 攻略型 |
| 热血传奇 | ❌ 无 | 间接承接 | 热血传奇 → 推荐盒子里的传奇0.1折手游 | 截流型 |
| 三国杀 | ✅ 有 | 直接承接 | 三国杀攻略 → 盒子折扣页 | 攻略型 |

#### 4.3 盒子库存的正确用法

盒子库存不是"能写什么"的限制，而是"转化时推荐什么"的素材库：

```
错误：盒子里有69款传奇游戏 → 所以做传奇站
正确：传奇品类搜索量大 → 所以做传奇站 → 文章里推荐盒子里有的传奇游戏

错误：盒子里没有热血传奇 → 所以不能写热血传奇
正确：热血传奇搜索量大 → 写热血传奇截流文章 → 推荐盒子里有的传奇0.1折手游

错误：7仓位的主打游戏必须都在盒子里
正确：7仓位 = 搜索需求最大的7条词路 → 盒子有的直接转化，没有的截流转化
```

### 阶段 5：7仓位主打游戏选择

#### 5.1 仓位分配

从阶段2的搜索需求表中选7款，遵循以下优先级：

```
优先级1：S级衰退期大游（搜索量大、竞争退潮）→ 截流型仓位
优先级2：A级中热度同类（搜索量稳定、竞争适中）→ 攻略型仓位
优先级3：盒子里有的高折扣游戏 → 转化型仓位
```

**7仓位分配建议：**

| 仓位数量 | 游戏类型 | 内容策略 | 示例 |
|---|---|---|---|
| 2~3个 | S级衰退期大游（盒子可能没有） | 截流型：用大游戏名做钩子，推荐盒子里有的同类 | 阴阳师、率土之滨 |
| 3~4个 | A级中热度同类（盒子最好有） | 攻略型：直接写攻略，链盒子转化 | 少年三国志2、神仙道3、三国杀 |
| 1个 | 盒子里折扣最高的游戏 | 充值避坑型：直接对比官服 vs 盒子价差 | 盒子里的0.1折三国卡牌 |

**示例——三国卡牌子站7仓位分配：**

| 仓位 | 游戏 | 分级 | 盒子有 | 内容策略 | 承接方式 |
|---|---|---|---|---|---|
| 1 | 阴阳师 | S | ❌ | 截流型：阴阳师攻略 → 推荐二次元0.1折 | 间接承接 |
| 2 | 率土之滨 | S | ❌ | 截流型：SLG攻略 → 推荐三国SLG折扣 | 间接承接 |
| 3 | 少年三国志2 | A | ✅ | 攻略型：阵容/零氪/版本 | 直接承接 |
| 4 | 神仙道3 | A | ✅ | 攻略型：阵容/零氪/FAQ | 直接承接 |
| 5 | 三国杀 | A | ✅ | 攻略型：卡组/策略/FAQ | 直接承接 |
| 6 | 少年三国志零 | A | ✅ | 攻略型：与少年三国志2对比 | 直接承接 |
| 7 | 三国志幻想大陆（0.1折） | B | ✅ | 转化型：官服vs盒子价格对比 | 直接承接 |

#### 5.2 确认扩展游戏池

7仓位主打游戏之外，再确认一批扩展游戏池，用于：

- 截流文章中的"被推荐对象"
- 品类聚合导航文章的收录范围
- 后续仓位轮换的候选

扩展游戏池来源：
1. 盒子里有的同类游戏（优先，有直接转化路径）
2. 搜索量中等但竞争低的同类游戏
3. 新游（公测前1~2周截流）

#### 5.3 仓位间不互相竞争检查

7个仓位之间不能出现：
- 同一个搜索词被两个仓位同时做
- 两个仓位的文章标题高度相似
- 同一款游戏占据多个仓位（除非是不同专服版本，且搜索意图不同）

## 五、常见错误

| 错误 | 正确做法 |
|---|---|
| 从盒子库存倒推选题 | 从搜索需求出发选题 |
| 只做盒子里有的游戏 | 用热门游戏做钩子，盒子里有的做转化 |
| 7仓位全是盒子小游戏 | S级大游做截流 + A级中游做攻略 + 盒子高折扣做转化 |
| 认为盒子没有就不能写 | 截流型文章的核心就是写盒子没有的热门游戏 |
| 一上来就做"大品类词" | 先从具体游戏名+意图词切入，再做品类聚合 |
| 选了搜索量高但竞争全是大站的词 | 优先选竞争退潮的衰退期大游 |
| 7仓位选了7款同类同质游戏 | 7仓位要覆盖不同搜索意图（阵容/零氪/版本/截流） |

# 单站7仓位排期模板

> 用途：给单个候选子站排 7 仓位起步节奏，记录仓位定义、周排期和阶段判断。
> 维护规则：一站一份；每个仓位都要能追到自己的关键词池、后台观察标签和观察记录。公开 sitemap 仍按站点原有板块 / 分类输出。

---

## 一、怎么用

1. 先在 [候选子站登记表.md](./候选子站登记表.md) 建好这个站的登记卡。
2. 复制第二节的“单站 7 仓位排期模板”，新建一份当前站专用排期表。
3. 每天先按 M6 选出今天的 5 篇，再把“当前阶段 / 池子配比 / 页面类型 / 发布结果”回填到排期表。
4. 每周补一次“本周判断”；如果阶段从起步期切到观察期或赢家期，同步更新排期表里的默认配比。
5. 如果某个仓位冒头，只给这个仓位加档期，不直接打乱整站节奏。

## 三、首轮发布硬规则

```text
1. 每个仓位首轮必须有完整页面组：1个入口页 + 3个问题页 + 1个FAQ/对比页。
2. 起步期单站每天默认按 3:1:1 发；进入观察期后改成 3:2:0；赢家期改成 2:2:1。
3. 如果测试位当天没有合格任务，用增量池补位；如果 FAQ/对比页当天没有合格任务，用问题/攻略页补位。
4. 仓位只做后台赛马标签，不要因为换仓频繁改公开 sitemap 归属；公开结构仍以板块 / 分类为准。
5. 每篇页面都必须同时具备：站内主题页回链 + 主站目标页导流。
6. 先给 A-G 每个仓位一条曲线，再决定谁拿额外档期。
7. 观察期内只做偏置，不因为几天没起量就整站推倒重来。
```

---

## 四、冒头信号速记

```text
仓位级：14~21 天内出现稳定 Impression / Query / CTR 抬升
站级：30~60 天内已形成“主题页 → 问题页 → FAQ/对比页 → 主站目标页”闭环
可以加档期：改稿成本低、导流质量好、还能继续扩当前类目
不要加档期：边界混乱、词路和其他站重叠、文章虽然有收录但不送点击
```

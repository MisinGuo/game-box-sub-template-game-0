## 二、主站内容战略（SITE_ID = 1）

> ⚠️ **主站内容原则：功能型为主，不做深度SEO主力**
> 主站文章的作用是**服务产品页面**（游戏详情/盒子页），推动用户转化。深度攻略、卡组/教程类文章统一发到**子站（SITE_ID=8）**。
>
> **⚠️ 主站请迁移至子站（SITE_ID=8）发布的内容类型：**
> - 游戏深度攻略（新手教程 / 阵容推荐 / 版本分析 / 上分技巧）
> - 卡组推荐 / Deck Building / Meta 分析类文章

> - 任何与子站关键词重叠的内容
>
> 以上内容发到主站 = 与子站关键词竞争，两边权重都会下降；迁移子站后内容仸值不变。

| 级别 | 特征 | 示例游戏 | 截流策略 |

#### 痛点型文章结构（AI 框架）

```
1. 痛点共鸣（30字）：直接说出用户的困境，建立信任
2. 筛选标准说明（50字）：告诉用户"我们怎么衡量这个问题"
3. 推荐游戏列表（主体）：每款游戏聚焦回答该痛点
   - 不肝型：每日任务时长 / 离线收益机制
```

## 六、如何区分同名 Section（主站 vs 子站）

两个站目前使用**相同的 section slug 名称**，后台通过 **SITE_ID** 区分归属：

- 主站文章选 SITE_ID = `1`
- 子站文章选 SITE_ID = `8`

如需将来彻底隔离（子站用独立 slug），在子站 `wrangler.toml` 的 `[vars]` 中加入：

```toml
NEXT_PUBLIC_SECTION_SLUGS_GUIDES = "kapai-guide-section"
NEXT_PUBLIC_SECTION_SLUGS_REVIEWS = "kapai-review-section"
NEXT_PUBLIC_SECTION_SLUGS_NEWS = "kapai-news-section"
```

加完后子站只抓 `kapai-*` 前缀的 section，无需修改任何代码。

## 六、如何区分同名 Section（主站 vs 子站）

两个站目前使用**相同的 section slug 名称**，后台通过 **SITE_ID** 区分归属：

- 主站文章选 SITE_ID = `1`
- 子站文章选 SITE_ID = `8`

如需将来彻底隔离（子站用独立 slug），在子站 `wrangler.toml` 的 `[vars]` 中加入：

```toml
NEXT_PUBLIC_SECTION_SLUGS_GUIDES = "kapai-guide-section"
NEXT_PUBLIC_SECTION_SLUGS_REVIEWS = "kapai-review-section"
NEXT_PUBLIC_SECTION_SLUGS_NEWS = "kapai-news-section"
```

加完后子站只抓 `kapai-*` 前缀的 section，无需修改任何代码。

## 七、发布前检查清单

每篇文章发布前逐项确认：

### 后台配置（必填，填错文章不展示）

- [ ] **SITE_ID** 选对：主站 = `1`，子站 = `8`
- [ ] **Section Slug** 选对（参考四、快速发文对照表）
- [ ] ⚠️ `event-section`（专题）**仅主站有效**，子站没有对应前端页面，勿填

### SEO 字段（影响搜索排名）

- [ ] **标题**包含目标关键词（游戏名 + 意图词，如"怎么玩""0.1折""盒子"）
- [ ] **摘要/描述**已填写（前端用作 meta description，建议 80~120 字）
- [ ] **封面图**已上传（用作 og:image，缺失会影响社交分享展示）
- [ ] **关键词**已填写（后台表单「关键词」字段，逗号分隔；用于 `/tag/[tag]` 聚合页和 JSON-LD keywords；至少填游戏名 + 品类词 + 意图词）

### 内链（影响转化，每篇必须执行）

- [ ] 已链接对应**游戏详情页** `/games/[id]`
- [ ] 资讯/专题类已链接 `/01zhe/` 或 `/rank/`
- [ ] 转化型/问答型已链接**盒子页** `/boxes/[id]`
- [ ] 子站转化型已链接**主站盒子页** `https://www.5awyx.com/boxes/[id]`
- [ ] 推荐了 3~5 篇相关文章（站内循环）

### 内容质量

- [ ] AI 生成内容已人工审核，无明显错误/幻觉数据
- [ ] 折扣数据、礼包内容真实可验证（不夸大）
- [ ] 安全顾虑型文章结论导向正规盒子，不引发不安情绪
- [ ] 礼包码大全类文章：码已验证有效，下次更新日期已记录

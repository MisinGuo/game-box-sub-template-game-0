# M5：文章角度矩阵

> **这份材料回答：** 当前站点当前仓位，可以写哪些任务卡？
> **输入：** 当前站点信息 + M1 游戏池 + M2 `keywords` + M3 `pain_points`
> **输出：** 带池子和页面类型标记的任务卡矩阵
> **后续用途：** M6 今日任务筛选的 `task_matrix` 输入
> **边界提醒：** 这一步不重新分池，只把 M1 已确认的池子归属展开成任务卡。

---

## 输出格式（标准化）

```
| site_key | 仓位 | 游戏池类型 | 页面类型 | 角度类型 | 游戏名 | 文章标题 | 核心论点 | 对应搜索词 |
|----------|------|------------|----------|----------|--------|----------|----------|------------|
| card-a | A | 日常池 | 入口/主题页 | 平替推荐型 | 率土之滨 | 2026年类似率土之滨的SLG手游推荐，这3款上班族最爱 | 低肝度平替推荐 | 类似率土之滨的游戏 |
| card-a | A | 日常池 | 问题/攻略页 | 痛点解决型 | 率土之滨 | 率土之滨太肝了？这几款国战SLG每天10分钟就够 | 时间压力解决方案 | 比率土之滨肝度低的游戏 |
| card-a | A | 增量池 | FAQ/对比页 | 充值避坑型 | 率土之滨 | 率土之滨月卡值不值？老玩家转向这款打折更多 | 首充性价比对比 | 率土之滨充值值不值 |
```

8 种角度类型：截流型 / 痛点解决型 / 充值避坑型 / 平替推荐型 / 入坑决策型 / 零氪攻略型 / 时间友好型 / 跑路停服型

页面类型只分 3 类：入口/主题页、问题/攻略页、FAQ/对比页。

---

## 方案一：AI 生成

**适用：** 正常流程，基于 M1+M2+M3 的真实输入生成

### 系统提示词

```
你是一名 SEO 内容策划，专门为手游推广站生产高转化文章选题。

⚠️ 重要规则：
1. 每篇文章的标题核心词必须来自用户输入的"真实搜索关键词"中的某个词，不得自行发明搜索词。
2. `游戏池类型` 必须沿用用户输入里的已确认值，不得自行升池、降池或改池。
3. `仓位` 只能沿用当前站点既有的 A-G 仓位，不得新建仓位或跨站重排。

请根据用户提供的当前站点信息、游戏池类型、关键词表、痛点清单，生成当前仓位下可写的任务卡，并给每条任务标记：

- `site_key`
- `仓位`
- `游戏池类型`（日常池 / 增量池 / 测试位）
- `页面类型`（入口/主题页 / 问题/攻略页 / FAQ/对比页）

按以下 8 类分别列出角度：

1. 【截流型】标题含"平替/同类/喜欢XX的玩家"，目标拦截搜索该游戏的用户
2. 【痛点解决型】标题含"太肝/太氪/不想花钱/上班族能玩吗"，精准命中流失意图
3. 【充值避坑型】标题含"月卡/首充/充值值不值/省钱方案"，拦截消费决策
4. 【平替推荐型】标题含"有没有类似/更好玩的同类游戏"，收割换游意图
5. 【入坑决策型】标题含"值得玩吗/好玩吗/2026年还能玩吗"，帮助观望用户决策
6. 【零氪攻略型】标题含"零氪玩到哪步/不充钱能玩多久"，服务低付费用户
7. 【时间友好型】标题含"上班族/每天几分钟/不用花太多时间"，收割时间受限用户
8. 【跑路/停服型】标题含"还值得玩吗/会停服吗/稳不稳"，拦截犹豫迁移的玩家

每类标题要求：
- 标题中必须代入真实的游戏名
- 标题核心词必须来自输入的搜索词（直接包含或自然变体）
- 同时说明：① 核心论点（不超过20字）② 对应的搜索关键词
- 当前仓位的输出结构尽量满足：至少 1 条入口/主题页任务、至少 1 条 FAQ/对比页任务，其余主力为问题/攻略页
- 测试位的任务不超过全部输出的 20%
- 起步期日常池任务应明显多于增量池和测试位任务

输出格式（表格）：
| site_key | 仓位 | 游戏池类型 | 页面类型 | 角度类型 | 游戏名 | 文章标题 | 核心论点 | 对应搜索词 |
|----------|------|------------|----------|----------|--------|----------|----------|------------|
```

### 用户提示词

```
当前站点 site_key：${current_site_key}
当前仓位：${current_slot}
当前阶段：${current_phase}
当前游戏池类型：${pool_type}
游戏名：${game_name}

真实搜索关键词（来自 M2）：
${keywords}

痛点清单（来自 M3）：
${pain_points}
```

| 变量 | 含义 | 来源 |
|------|------|------|
| `current_site_key` | 当前站点标识 | 候选子站登记表 |
| `current_slot` | 当前执行仓位 | 单站7仓位排期模板 |
| `current_phase` | 当前阶段 | 起步期 / 观察期 / 赢家期 |
| `pool_type` | 当前游戏所属池子 | M1 复核后的池子归属 |
| `game_name` | 截流目标游戏名 | M1 输出 |
| `keywords` | 完整关键词表格 | M2 输出（直接复制粘贴） |
| `pain_points` | 痛点清单 | M3 输出（直接复制粘贴） |

## 方案二：人工规划

**适用：** 对游戏有深入了解，想自己控制文章方向

对照 8 种角度类型，手动按输出格式填写，核心要求：
- 每个标题包含 M2 中真实存在的关键词
- 不同角度的核心论点不能重复
- 每条任务都必须继承已确认池子，并标记页面类型
- 同一批任务里要同时出现入口/主题页、问题/攻略页、FAQ/对比页三类页面

## 角度类型速查

> 选题/内链决策时快速查阅。

| 角度类型 | 目标用户 | 转化点 | 更适合的页面类型 |
|----------|----------|--------|------------------|
| 截流型 | 正在玩/搜大热门游戏的用户 | 同类游戏推荐 → 子站攻略 | 入口/主题页 或 问题/攻略页 |
| 痛点解决型 | 已经不满当前游戏的用户 | 解决痛点的替代游戏 | 问题/攻略页 |
| 充值避坑型 | 准备充钱的用户 | 官服 → 盒子首充转化 | FAQ/对比页 |
| 平替推荐型 | 主动在找换游方向的用户 | 推荐列表 → 子站攻略 | 入口/主题页 |
| 入坑决策型 | 观望中、未决策的用户 | 打消顾虑 → 降低门槛 | 问题/攻略页 |
| 零氪攻略型 | 不想花钱但想玩的用户 | 推动下载 + 盒子注册礼包 | 问题/攻略页 |
| 时间友好型 | 轻度玩家、上班族 | 推荐低时间成本游戏 | 问题/攻略页 |
| 跑路/停服型 | 担心游戏稳定性的用户 | 推荐稳定运营游戏 | FAQ/对比页 |

---

## 质量检验

- [ ] 每个标题是否包含 M2 中真实存在的关键词（或其自然变体）
- [ ] 任务卡是否已标记 `site_key / 仓位 / 游戏池类型 / 页面类型`
- [ ] 同一批任务中是否至少有 1 条入口/主题页和 1 条 FAQ/对比页任务
- [ ] 测试位任务是否没有超过这一批任务的 20%
- [ ] 标题是否包含年份（2026）或具体钩子词

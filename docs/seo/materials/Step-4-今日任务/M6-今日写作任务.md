# M6：今日写作任务

> **这份材料回答：** 当前站点今天这 5 篇应该怎么分？
> **输入：** 当前站点信息 + M5 `task_matrix`（所有游戏的任务卡）+ 已发布去重索引
> **输出：** 当前站点今日 5 个带顺序的写作任务
> **后续用途：** 每个任务单独进入 M8 生成完整文章

> **边界提醒：** 这一步只能在已确认 `site_key / 仓位 / 游戏池类型` 的任务库里筛选，不负责重分池，也不负责重定今天执行哪个仓位。

---

## 输出格式（标准化）

```
| 顺序 | site_key | 仓位 | 游戏池类型 | 页面类型 | 游戏名 | 角度类型 | 目标关键词 | 推荐标题 | 写作重点提示 |
|------|----------|------|------------|----------|--------|----------|------------|----------|--------------|
| 1 | card-a | A | 日常池 | 入口/主题页 | 率土之滨 | 平替推荐型 | 类似率土之滨的游戏 | 2026年类似率土之滨的SLG手游... | 先把当前仓位的主题词路讲清楚 |
| 2 | card-a | A | 日常池 | 问题/攻略页 | 率土之滨 | 痛点解决型 | 比率土之滨肝度低的游戏 | 率土之滨太肝了？... | 重点突出低肝度与上班族场景 |
```

去重索引格式（存入素材卡）：
```
card-a|率土之滨|截流型
card-a|率土之滨|痛点解决型
card-a|原神|充值避坑型
```

---

## 方案一：AI 筛选

**适用：** 任务库积累了多款游戏后，让 AI 按配比规则自动选

### 系统提示词

```
你是一名内容运营，负责从手游攻略站的文章待写任务库中，每日挑选最高价值的任务来生产。

请根据用户提供的任务库、当前阶段和已发布去重索引：
1. 只允许从已确认了 `site_key / 仓位 / 游戏池类型` 的任务卡中选；凡是 AI 建议未确认、跨站或跨仓位任务一律排除
2. 排除已发布的 `site_key|游戏名|角度类型` 组合
3. 必须先满足“池子配比”和“页面类型配比”，再谈优先级
4. 不得为了分数高临时改今天的 `current_slot`，也不得把测试位任务抬成日常池
5. 最终只输出今天最值得写的 5 篇，顺序从 1 到 5

池子配比规则：
- 候选子站起步期：3 篇日常池 + 1 篇增量池 + 1 篇测试位
- 候选子站观察期：3 篇日常池 + 2 篇增量池 + 0 篇测试位
- 赢家子站加码期：2 篇日常池 + 2 篇增量池 + 1 篇更大词/更大游戏测试位

页面类型配比规则：
- 默认保持 1 篇入口/主题页 + 3 篇问题/攻略页 + 1 篇 FAQ/对比页

回填规则：
- 如果测试位当天没有合格任务，用增量池补位
- 如果 FAQ/对比页当天没有合格任务，用问题/攻略页补位
- 如果入口/主题页当天没有合格任务，优先从日常池选一个聚合型任务补位

优先级规则（流量 × 转化 双维度）：
1. 有明确停服/回归/联动事件触发 → 最高优先（时效窗口短，立刻写）
2. 关键词有搜索量 且 为截流型/平替推荐型 → 高优先（流量×转化双赢）
3. 关键词有搜索量 且 为充值避坑型 → 高优先（商业转化意图极强）
4. 痛点解决型（对应关键词在搜索词表中能找到）→ 中优先
5. 入坑决策型 + 零氪攻略型 → 正常优先
6. 无对应搜索词的时间友好型/跑路停服型 → 备选

输出格式（5行表格）：
| 顺序 | site_key | 仓位 | 游戏池类型 | 页面类型 | 游戏名 | 角度类型 | 目标关键词 | 推荐标题 | 写作重点提示 |
|------|----------|------|------------|----------|--------|----------|------------|----------|--------------|
```

### 用户提示词

```
待写任务库（各游戏 M5 输出汇总）：
${task_matrix}

已发布去重索引（格式：site_key|游戏名|角度类型，每行一条；无则填"暂无"）：
${published_index}

当前站点 site_key（必填）：${current_site_key}
当前阶段（必填）：${current_phase}
当前仓位（必填）：${current_slot}
当前站点状态（必填）：${current_site_status}
当前仓位状态（必填）：${current_slot_status}

今天的游戏偏向（不限则填"不限"）：${today_preference}
```

| 变量 | 含义 | 来源 |
|------|------|------|
| `task_matrix` | 多款游戏角度矩阵合并 | 复制各游戏 M5 输出表格 |
| `published_index` | 当前站点已发布文章索引 | 从内容追踪矩阵提取当前 `site_key` 的 ✅ 条目 |
| `current_site_key` | 今天执行的站点标识 | 候选子站登记表 |
| `current_phase` | 今天执行的阶段 | 起步期 / 观察期 / 赢家期 |
| `current_slot` | 今天执行的仓位 | 单站7仓位排期模板 |
| `current_site_status` | 当前站点状态 | Candidate / Winner / Loser / Frozen |
| `current_slot_status` | 当前仓位状态 | Running Slot / Winner Slot / Weak Slot |
| `today_preference` | 今天想写的题材方向（可选） | 三国题材 / 不限 |

## 方案二：程序自动筛选

**适用：** 任务库大（50篇以上积累），想完全自动化

**优先级评分规则实现：**

```python
PRIORITY_SCORES = {
    "截流型": 90,
    "平替推荐型": 85,
    "充值避坑型": 85,
    "痛点解决型": 70,
    "入坑决策型": 60,
    "零氪攻略型": 55,
    "时间友好型": 40,
    "跑路停服型": 35,
}

SEARCH_VOLUME_BONUS = {"高": 30, "中": 15, "低": 0, "待查": 5}

POOL_QUOTAS = {
    "起步期": {"日常池": 3, "增量池": 1, "测试位": 1},
    "观察期": {"日常池": 3, "增量池": 2, "测试位": 0},
    "赢家期": {"日常池": 2, "增量池": 2, "测试位": 1},
}

PAGE_QUOTAS = {"入口/主题页": 1, "问题/攻略页": 3, "FAQ/对比页": 1}

def score_task(task, published_index):
    key = f"{task['site_key']}|{task['游戏名']}|{task['角度类型']}"
    if key in published_index:
        return -1
    if task.get("池子确认状态") not in {None, "已确认", "人工确认", "程序判定"}:
        return -1
    base = PRIORITY_SCORES.get(task["角度类型"], 30)
    volume_bonus = SEARCH_VOLUME_BONUS.get(task.get("搜索量级", "待查"), 5)
    return base + volume_bonus

def select_daily_tasks(all_tasks, published_index, current_site_key, current_phase, current_slot, today_preference=None, n=5):
    slot_tasks = [
        task for task in all_tasks
        if task.get("site_key") == current_site_key and task.get("仓位") == current_slot
    ]
    scored = [(task, score_task(task, published_index)) for task in slot_tasks]
    scored = [(t, s) for t, s in scored if s >= 0]

    quotas = POOL_QUOTAS[current_phase]
    selected = []
    for pool_type, limit in quotas.items():
        pool_candidates = [(t, s) for t, s in scored if t.get("游戏池类型") == pool_type]
        pool_candidates.sort(key=lambda x: x[1], reverse=True)
        selected.extend([t for t, _ in pool_candidates[:limit]])

    final_tasks = rebalance_by_page_type(selected, PAGE_QUOTAS)
    return final_tasks[:n]
```

## 方案三：人工选定

**适用：** 任务少（10篇以内积累），直接看表格选

1. 先确认今天执行的 `site_key / 当前阶段 / 当前仓位`
2. 再确认当前站点状态和当前仓位状态，不要让 AI 代替双层赛马做决策
3. 打开该站点该仓位对应的 M5 输出任务卡
4. 先按配比挑：起步期 3:1:1；观察期 3:2:0；赢家期 2:2:1
5. 再按页面类型修正成 1 篇入口/主题页 + 3 篇问题/攻略页 + 1 篇 FAQ/对比页
6. 最后按优先级规则排顺序，填入今日写作任务表格

## 更新去重索引

每次写完一篇文章，立即在内容追踪矩阵中更新：

```
素材卡「已完成的文章角度」栏：
✅ 截流型 | 2026-04-23
```

下次运行 M6 时，把当前站点所有 ✅ 条目提取成 `site_key|游戏名|角度类型` 格式粘给 `published_index`，避免同站重复选题。

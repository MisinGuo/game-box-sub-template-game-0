# M7：子站游戏卡片库

> **这份材料回答：** 文章里可以推荐哪些游戏？每款游戏的详情是什么？
> **输入：** 子站数据库（平台真实数据）
> **输出：** 可推荐游戏的卡片信息，供 M8 文章生成使用
> **⚠️ 重要：** 这份材料**不能**用 AI 生成，必须来自子站真实数据。AI 无法知道你的子站收录了哪些游戏、折扣是多少。

---

## 为什么不能 AI 生成

AI 如果自行推荐游戏，会出现：
- 推荐子站根本没收录的游戏（内链失效）
- 编造不存在的折扣数字（影响信任度）
- 引导用户跳转到不存在的页面（转化为零）

**数据只能来自平台，这是整个流程中唯一不可替代的来源。**

---

## 输出格式（标准化）

```
游戏名 | 一句话定位 | 核心卖点（针对目标痛点） | 子站详情页路径 | 主站盒子路径 | 折扣信息
```

示例：
```
--- 三国题材 ---
三国志战棋版 | 回合制策略，SLG轻度玩法 | 每日任务15分钟，无公会强制 | /games/123 | /boxes/456 | 首充0.1折
云上城之歌 | 武将卡牌收集，无强制PVP | 零氪武将获取路径清晰 | /games/124 | /boxes/457 | 首充5折
```

## 获取方式一：查询数据库（程序）

**最推荐，数据最准确**

```sql
-- 查询子站已收录、按题材分组的游戏卡片
SELECT 
    g.game_name,
    g.short_description AS one_liner,
    g.selling_points,
    CONCAT('/games/', g.id) AS game_url,
    CONCAT('/boxes/', b.id) AS box_url,
    b.discount_info,
    g.genre_tag
FROM games g
LEFT JOIN boxes b ON b.game_id = g.id
WHERE g.site_id = 8  -- 子站 ID
  AND g.status = 'published'
ORDER BY g.genre_tag, g.hot_score DESC;
```

把查询结果按题材分组，粘贴到模板中即可。

## 获取方式二：人工维护卡片库

**适用：** 没有直接数据库访问权限，或只有少量游戏

按以下格式手动建立并维护，每次子站新增游戏时更新一次：

```
--- 三国题材 ---
[游戏名A] | [一句话定位] | [核心卖点] | /games/[ID] | /boxes/[ID] | [折扣:X折]

--- 二次元题材 ---
[游戏名D] | [一句话定位] | [核心卖点] | /games/[ID] | /boxes/[ID] | [折扣:X折]

--- 西游/忍者题材 ---
[游戏名F] | [一句话定位] | [核心卖点] | /games/[ID] | /boxes/[ID] | [折扣:X折]
```

> 折扣数字来自主站后台实际价格，不接受估算。

## 获取方式三：调用子站 API（程序）

**适用：** 子站有开放接口

```python
import requests

def fetch_games_by_genre(site_id: int, genre: str):
    """从子站 API 获取指定题材的已收录游戏"""
    resp = requests.get(
        f"https://api.your-site.com/games",
        params={"site_id": site_id, "genre": genre, "status": "published"},
        headers={"Authorization": "Bearer YOUR_API_TOKEN"},
        timeout=10,
    )
    resp.raise_for_status()
    games = resp.json()["data"]
    
    cards = []
    for g in games:
        cards.append({
            "game_name": g["name"],
            "one_liner": g["short_description"],
            "selling_points": g["selling_points"],
            "game_url": f"/games/{g['id']}",
            "box_url": f"/boxes/{g['box_id']}" if g.get("box_id") else None,
            "discount": g.get("discount_info", "[折扣:X折待填]"),
        })
    return cards

# 使用示例
san_guo_games = fetch_games_by_genre(site_id=8, genre="三国")
```

## 在 M8 中使用

从卡片库中，根据截流目标游戏的题材，选 **2~3 款**同题材游戏复制到 M8 的 `game_list` 变量中。

选择原则：
1. 卖点要与截流目标游戏的痛点形成对比（如目标游戏「太肝」，就选「低肝度」的游戏）
2. 有首充折扣优先（转化率更高）
3. 不选运营状况不明或可能停服的游戏

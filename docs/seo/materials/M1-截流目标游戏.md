# M1：目标游戏池规划（原截流目标游戏）

> **这份材料回答：** 当前站点这一轮内容生产，要把哪些游戏放进日常池、增量池和测试位？
> **输入：** 当前站点的小类目方向 + 当前阶段 + 当前仓位重点
> **输出：** 一份带池子归属的目标游戏池列表
> **后续用途：** M2、M3、M4 的 `game_name` 输入；M5/M6 的游戏池配比依据

---

## 输出格式（标准化，供后续材料使用）

```
| 游戏名 | 品类（细分） | 入池原因 | 截流/承接角度建议 | 建议池子 | 建议仓位 |
|---|---|---|---|---|---|
| 率土之滨 | SLG 策略 | 搜索量稳定、流失意图明显 | 主打「上班族不想铺路的平替」 | 日常池 | A |
| ... | ... | ... | ... | ... | ... |
```

池子定义：

- 日常池：当前站点要持续拿数据的主力游戏。每天内容生产的核心盘。
- 增量池：同小类目下准备继续扩量的候补游戏。不是陌生新游，但还没进入主力盘。
- 测试位：新游戏、新上线词、或刚并入的小候补游戏。每天只给很少资源试错。

默认数量建议：

```text
候选子站起步期：日常池 3~5 个游戏；增量池 2~3 个游戏；测试位 1~2 个游戏
候选子站观察期：日常池 4~6 个游戏；增量池 2~4 个游戏；不再固定每天上测试位
赢家子站加码期：日常池 3~5 个游戏；增量池 3~5 个游戏；更大词/更大游戏测试位 1~2 个
```

## 方案一：AI 生成

**适用：** 快速开始，不需要查资料

### 系统提示词

```
你是一名手游市场分析师，熟悉国内安卓手游的生命周期规律和玩家社区动态。

请根据用户提供的当前站点信息，列出适合这个站点的小类目游戏池，并按「日常池 / 增量池 / 测试位」三类分配。

筛选标准如下：
1. 必须和当前站点的小类目一致，不能越界到其他子站的主题边界
2. 日常池优先放搜索需求稳定、可连续写 2~4 周的游戏
3. 增量池优先放同类目下准备继续扩词的候补游戏
4. 测试位只放 1~2 个新游戏、新上线词或刚并入的小候补游戏

输出每款游戏：
游戏名 | 品类（细分） | 入池原因（一句话） | 截流/承接角度建议 | 建议池子 | 建议仓位

要求：
- 只列用户指定的小类目
- 优先列当前阶段最适合服务日常盘的游戏，不要全塞进测试位
- 截流/承接角度建议要具体（例如：主打「上班族肝度低的平替」方向）
- 候选子站起步期默认按 3:1:1 去反推池子规模；观察期按 3:2:0；赢家期按 2:2:1

输出格式（表格）：
| 游戏名 | 品类（细分） | 入池原因 | 截流/承接角度建议 | 建议池子 | 建议仓位 |
|---|---|---|---|---|---|
```

### 用户提示词

```
当前站点 site_key：${current_site_key}
当前站点小类目：${category}
当前阶段：${current_phase}
当前优先仓位：${current_slot}
```

| 变量 | 含义 | 示例 |
|------|------|------|
| `current_site_key` | 当前站点标识 | card-a |
| `category` | 子站定位的小类目 | 零氪卡牌/放置卡牌 |
| `current_phase` | 当前阶段 | 起步期 / 观察期 / 赢家期 |
| `current_slot` | 当前优先仓位 | A |

## 方案二：人工选定

**适用：** 已经有想法、或想精确控制选题方向

1. 先查当前站点的候选子站登记表和单站排期，确认今天服务哪个小类目、哪个仓位
2. 打开 TapTap → 分类 → 选目标品类 → 按「热度」排序
3. 筛选条件：评价数 10 万+ 但近期更新减少 / 差评率上升
4. 参考贴吧/B站UP主动态：哪些游戏玩家在大量发「弃坑帖」
5. 按照输出格式手动填表，并标记每款游戏属于日常池 / 增量池 / 测试位

## 方案三：程序生成

**适用：** 批量处理、定期自动更新候选列表

**思路：** TapTap 游戏列表按热度排序，抓取评分 + 近期评论量变化，再按当前阶段分配进池子

```python
# 伪代码示意
# 1. 抓取 TapTap 指定小类目 Top 100 游戏
# 2. 对比 30天前/90天前 的评论增量与搜索活跃度
# 3. 按当前阶段分配：稳定需求 → 日常池；可扩词路 → 增量池；新对象 → 测试位
# 4. 输出标准格式表格

games = fetch_taptap_games(category="零氪卡牌")
stable = [g for g in games if g.review_growth_30d > -0.2 and g.search_trend == "stable"]
incremental = [g for g in games if g.keyword_expansion_room == "high"]
test_candidates = [g for g in games if g.is_new_release or g.is_new_keyword]
```

> **注意：** TapTap 无公开 API，需用 requests + HTML 解析，或使用浏览器自动化。

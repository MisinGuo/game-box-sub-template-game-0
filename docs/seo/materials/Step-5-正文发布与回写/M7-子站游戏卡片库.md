# M7：子站游戏卡片库

> **这份材料回答：** 文章里可以推荐哪些游戏？每款游戏的详情是什么？
> **输入：** 当前候选子站或赢家子站数据库（平台真实数据）
> **输出：** 可推荐游戏的卡片信息，供 M8 文章生成使用
> **⚠️ 重要：** 这份材料**不能**用 AI 生成，必须来自子站真实数据。AI 无法知道你的子站收录了哪些游戏、折扣是多少。
> **⚠️ 更新规则：** 每次当前站点新增游戏或折扣变化时更新一次；box_id 需从后台确认。

---

## 为什么不能 AI 生成

AI 如果自行推荐游戏，会出现：
- 推荐子站根本没收录的游戏（内链失效）
- 编造不存在的折扣数字（影响信任度）
- 引导用户跳转到不存在的页面（转化为零）

**数据只能来自当前站点的平台数据，这是整个流程中唯一不可替代的来源。**

---

## 输出格式（标准化）

```
游戏名 | 一句话定位 | 核心卖点（针对目标痛点） | 子站详情页路径 | 主站盒子路径 | 折扣信息
```

## 在 M8 中使用

从卡片库中，根据截流目标游戏的题材，选 **2~3 款**同题材游戏复制到 M8 的 `game_list` 变量中。

选择原则：
1. 卖点要与截流目标游戏的痛点形成对比（如目标游戏「太肝」，就选「低肝度」的游戏）
2. 有首充折扣优先（转化率更高）
3. 不选运营状况不明或可能停服的游戏

## ⚠️ 待确认事项

以下字段需后端启动后通过 API 确认：

1. **box_id**：所有游戏的 `/boxes/[id]` 路径中的 `[id]` 需从 `/api/public/boxes` 接口获取
2. **折扣信息**：传奇类游戏的折扣标签多为 `[待确认]`，需从 `/api/public/games` 接口的 `discountLabel` 字段确认
3. **SITE_ID**：3个新子站的 SITE_ID 需在后台新建站点记录后分配

确认步骤：
1. 启动后端服务
2. 调用 `GET /api/public/boxes` 获取所有盒子数据
3. 调用 `GET /api/public/games?pageSize=10000&queryMode=related` 获取完整游戏数据
4. 用 `game_id` 匹配 `box_id`，更新卡片库中的 `/boxes/[id]` 和折扣信息

# M2：游戏关键词表

> **这份材料回答：** 玩家在搜索引擎里真实输入什么词来找这款游戏？
> **输入：** M1 的 `game_name`（游戏名）
> **输出：** 15~30 个真实搜索词，含搜索量级判断
> **后续用途：** M5 角度矩阵的 `keywords` 输入；M8 文章的 SEO 埋词依据

---

## 输出格式（标准化）

```
| 类型 | 关键词 | 预估搜索量级 |
|------|--------|--------------|
| 直接需求词 | 率土之滨好玩吗 | 中 |
| 直接需求词 | 类似率土之滨的游戏 | 高 |
| 痛点词 | 比率土之滨肝度低的游戏 | 中 |
| 痛点词 | 上班族能玩的SLG手游 | 中 |
| 竞品词 | 率土之滨同类游戏推荐 | 中 |
| 竞品词 | 率土之滨换游玩什么 | 低 |
```

搜索量级定义：高 = 月均万次+ / 中 = 月均千次 / 低 = 月均百次 / 待查

## 方案一：AI 生成

**适用：** 快速产出初稿，再人工验 2~3 个高量级词

### 系统提示词

```
你是一名 SEO 关键词研究员，熟悉百度、TapTap 的手游玩家搜索行为。

请根据用户给出的游戏名，生成用户在搜索引擎中真实可能输入的关键词，按三类分别给出 8~10 个关键词：

【直接需求词】用户搜索该游戏相关内容的变体，例如：
- [游戏名]好玩吗 / [游戏名]值得玩吗 / 类似[游戏名]的游戏 / [游戏名]平替推荐

【痛点词】用户对该游戏产生厌倦/不满时搜的词，例如：
- [游戏名]太肝了 / 比[游戏名]肝度低的游戏 / 上班族能玩的[题材]手游

【竞品词】以该游戏名为对比锚点的长尾词，例如：
- 类似[游戏名]的卡牌手游 / [游戏名]同类游戏推荐 / [游戏名]换游玩什么

要求：
1. 每个词必须是用户真实可能输入的自然语言（5~15字）
2. 标注预估搜索量级：高（月均万次+）/ 中（月均千次）/ 低（月均百次）/ 待查

输出格式（表格）：
| 类型 | 关键词 | 预估搜索量级 |
|------|--------|--------------|
```

### 用户提示词

```
目标游戏：${game_name}
```

| 变量 | 含义 | 来源 |
|------|------|------|
| `game_name` | M1 选定的截流目标游戏名 | 如：率土之滨 |

## 方案三：程序生成

**适用：** 批量处理多款游戏

**思路：** 调用百度 Suggest API 获取联想词，再用 AI 批量补充痛点词

```python
import requests

def get_baidu_suggest(keyword):
    """获取百度搜索框联想词"""
    url = "https://suggestion.baidu.com/su"
    params = {"wd": keyword, "cb": "callback", "from": "pc_web"}
    resp = requests.get(url, params=params, timeout=5)
    # 解析 JSONP 响应，提取 s 字段（建议词列表）
    # 示例响应: callback({"q":"rate土","p":false,"s":["率土之滨好玩吗","率土之滨换游","..."]})
    return parse_jsonp(resp.text)

def get_keywords_for_game(game_name):
    queries = [
        game_name,
        f"类似{game_name}",
        f"{game_name}平替",
        f"{game_name}好玩吗",
        f"比{game_name}肝度低",
    ]
    all_keywords = []
    for q in queries:
        all_keywords.extend(get_baidu_suggest(q))
    return deduplicate(all_keywords)
```

> 输出结果缺少搜索量数字，仍需人工用百度指数核实"中/高"量级词。

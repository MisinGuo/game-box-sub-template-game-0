# M4：游戏专属机制梗

> **这份材料回答：** 这款游戏的老玩家之间用什么专属词汇说话？
> **输入：** M1 的 `game_name`
> **输出：** 5~10 个机制名词 / 老玩家专属梗
> **后续用途：** M8 文章的 `game_mechanics` —— 这是消除「AI 通稿感」的关键素材

---

## 为什么这个材料最重要

没有专属机制词的文章，开头一看就是模板：

> ❌ 「率土之滨是一款经典的 SLG 手游，深受玩家喜爱。」

有了专属机制词，开头才有老玩家味道：

> ✅ 「在率土之滨打了三个征服赛季，铺路铺到深夜，联盟战打到凌晨，才发现自己一分钱没花到刀口上……」

---

## 输出格式（标准化）

```
游戏名：率土之滨

机制名词：征服赛季、铺路、占州、配将表、撞州、屯田
老玩家梗：「这游戏就是铺路游戏」「配将表三天一换」「联盟不活跃直接开除」
核心场景：大地图占领州郡、征服赛季限时推进、武将搭配策略
```

## 方案一：AI 生成

**适用：** 大热门游戏，AI 有足够训练数据

**⚠️ 风险：** AI 可能混淆相似游戏的术语，生成后必须核实

### 系统提示词

```
你是一名深度玩过 ${game_name} 的资深老玩家，对该游戏的系统机制和社区文化了如指掌。

请列出该游戏玩家社区中高频使用的专属词汇，包括：
1. 游戏内系统/机制名称（该游戏独有的，不是通用手游术语）
2. 玩家自创的简称/梗/黑话
3. 该游戏独有的核心玩法场景描述

输出格式：
机制名词：[词1]、[词2]、[词3]（至少5个）
老玩家梗：[「梗1」]、[「梗2」]（玩家真实说法）
核心场景：[一句话描述最典型的游戏场景，用于文章开篇]
```

### 用户提示词

```
游戏名：${game_name}
```

## 方案二：人工收集（推荐，15分钟以内）

**适用：** 保证准确性，尤其冷门游戏

| 来源 | 操作 |
|------|------|
| 百度贴吧 | 搜「[游戏名] 吧」→ 看高赞帖子标题，记录高频词 |
| TapTap 评论 | 搜游戏 → 好评/差评区 → 找玩家使用的非通用词汇 |
| B站评论区 | 搜「[游戏名] 攻略」→ 看弹幕和评论区的黑话 |
| 知乎 | 搜「[游戏名] 怎么玩」→ 看回答者使用的术语 |

收集完按输出格式整理即可。

## 方案三：程序辅助提取

**适用：** 有抓取能力，想自动化

**思路：** 抓取贴吧 / TapTap 评论，用 TF-IDF 或词频统计找出该游戏高频专属词

```python
from collections import Counter
import jieba

# 通用手游词汇（排除这些）
COMMON_GAMING_WORDS = {"日常", "活动", "氪金", "手游", "玩家", "游戏", "更新", "版本", "充值"}

def extract_game_specific_terms(reviews: list[str], top_n=20):
    """从评论列表中提取游戏专属高频词"""
    all_words = []
    for review in reviews:
        words = jieba.cut(review)
        all_words.extend([w for w in words if len(w) >= 2 and w not in COMMON_GAMING_WORDS])
    
    counter = Counter(all_words)
    # 取出现频率高但不在通用词表中的词
    return counter.most_common(top_n)

# 配合 M3 的评论抓取使用
reviews = fetch_taptap_reviews(game_id="12345")
terms = extract_game_specific_terms(reviews)
```

> 程序输出是候选词列表，还需要人工判断哪些是真正的「机制专属词」。

---

## 质量检验

- [ ] 每个词是否真的是这款游戏独有的（不是所有 SLG 都有的通用词）
- [ ] 是否包含该游戏最核心的游戏循环动词（如：铺路、占州）
- [ ] 是否有 1~2 个玩家自创的梗/黑话
- [ ] 核心场景是否能让老玩家一眼认出「这说的就是我」

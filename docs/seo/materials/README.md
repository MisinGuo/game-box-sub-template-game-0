# 内容生产材料库

> 把整个内容生产流程拆成 8 份独立材料，每份材料可以单独选择用 **AI 生成 / 人工填写 / 程序生成**。
> 材料之间存在依赖关系，但每份文件都是自包含的 —— 清楚说明需要什么、怎么产出、输出格式是什么。

---

## 材料依赖关系

```
M1: 截流目标游戏
    ↓ game_name
M2: 游戏关键词表 ──────────┐
    ↓ keywords             │
M3: 游戏痛点清单 ──────────┤
    ↓ pain_points          │
M4: 游戏专属机制梗          │
    ↓ game_mechanics        │
                           ↓
                    M5: 文章角度矩阵
                           ↓ task_matrix
M7: 子站游戏卡片库  →  M6: 今日写作任务
（平台数据，只读）          ↓ 单条任务
                    M8: 完整文章
                           ↓
                    M9: Meta Description
                           ↓
                    M10: 后台发布
                           ↓
                    M11: 追踪矩阵更新
```

## 哪些材料可以 100% AI 生成

| 材料 | AI 可信度 | 备注 |
|------|-----------|------|
| M1 截流目标游戏 | ✅ 可用 | AI 了解大热门游戏衰退情况 |
| M2 关键词表 | ⚠️ 需验证 | 搜索量级是估算，重要词要验证 |
| M3 痛点清单 | ⚠️ 需验证 | 冷门游戏可能编造，需用评论辅助版 |
| M4 机制梗 | ⚠️ 需验证 | AI 可能用错术语，建议人工核对 |
| M5 角度矩阵 | ✅ 可用 | 基于真实输入生成，质量稳定 |
| M6 今日任务 | ✅ 可用 | 或直接用程序按优先级规则筛选 |
| M7 游戏卡片 | ❌ 不可AI | 必须来自子站真实数据 |
| M8 完整文章 | ✅ 可用 | 需人工审核折扣数字和内链 |
| M9 Meta Description | ⚠️ 需微调 | AI 生成后人工核查字符数和钩子词位置 |
| M10 后台发布 | ❌ 不可AI | 操作性步骤，人工执行 或 程序调 API |
| M11 追踪矩阵更新 | ❌ 不可AI | 状态记录，人工维护 或 程序自动追加 |

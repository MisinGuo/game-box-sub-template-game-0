# 内容生产材料库

> 把整个内容生产流程拆成 11 份独立材料，每份材料可以单独选择用 **AI 生成 / 人工填写 / 程序生成**。
> 材料之间存在依赖关系，但每份文件都是自包含的 —— 清楚说明需要什么、怎么产出、输出格式是什么。

> 当前标准：所有子站类材料都先以 [候选子站登记表.md](../02-规划/候选子站登记表.md) 确认当前站点，不再默认固定某个 `SITE_ID=8`。

> 当前执行口径：单站每天固定 5 篇时，不按“5 个新游戏”来发，而按“日常池 / 增量池 / 新游戏测试位”来切。
> 候选子站起步期默认配比：`3:1:1`；观察期默认配比：`3:2:0`；赢家子站加码期默认配比：`2:2:1`。

> 决策权边界：`current_site_key / current_phase / current_slot / pool_type` 的最终值，必须先由候选子站登记表、单站 7 仓位排期表和最近观察结果确认。AI 在这套材料里只负责补候选、补标题、补排序，不负责改站点、改仓位、改池子。

> 结构口径：这里的 `current_slot / current_phase / page_type` 都是内部流程变量，用来做赛马控制和任务分发；公开页面结构与 sitemap 仍按站点原有板块 / 分类输出。

## 哪些材料可以 100% AI 生成

| 材料 | AI 可信度 | 备注 |
|------|-----------|------|
| M1 截流目标游戏 | ⚠️ 只可初筛 | AI 可以补候选，但最终分池必须看站级/仓位级赛马结果 |
| M2 关键词表 | ⚠️ 需验证 | 搜索量级是估算，重要词要验证 |
| M3 痛点清单 | ⚠️ 需验证 | 冷门游戏可能编造，需用评论辅助版 |
| M4 机制梗 | ⚠️ 需验证 | AI 可能用错术语，建议人工核对 |
| M5 角度矩阵 | ✅ 可用 | 前提是池子和仓位已确认，AI 只做任务展开 |
| M6 今日任务 | ⚠️ 有条件可用 | 只能在已确认 `site_key / slot / pool_type` 的任务库内筛选 |
| M7 游戏卡片 | ❌ 不可AI | 必须来自子站真实数据 |
| M8 完整文章 | ✅ 可用 | 需人工审核折扣数字和内链 |
| M9 Meta Description | ⚠️ 需微调 | AI 生成后人工核查字符数和钩子词位置 |
| M10 后台发布 | ❌ 不可AI | 操作性步骤，人工执行 或 程序调 API |
| M11 追踪矩阵更新 | ❌ 不可AI | 状态记录，人工维护 或 程序自动追加 |

## 单站 5 篇默认拆法

```text
起步期：3 篇日常池 + 1 篇增量池 + 1 篇新游戏测试位
观察期：3 篇日常池 + 2 篇增量池 + 0 篇新游戏测试位
赢家期：2 篇日常池 + 2 篇增量池 + 1 篇更大词/更大游戏测试位
```

页面类型配比默认保持：

```text
1 篇入口/主题页 + 3 篇问题/攻略页 + 1 篇 FAQ/对比页
```

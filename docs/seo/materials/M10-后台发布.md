# M10：后台发布

> **这份材料回答：** 文章怎么发布到后台？每个字段填什么？
> **输入：** M8 文章正文 + M9 Meta Description + M7 游戏/盒子 ID
> **输出：** 发布完成，文章在子站前台可访问
> **操作路径：** 后台 → 内容管理 → 文章列表 → 关联网站 SITE_ID=8 → 新增

---

## 发布字段清单

| 字段 | 值 / 填写规则 |
|------|--------------|
| 站点 | `SITE_ID = 8`（子站，kapai-web.94kj.cn） |
| 板块 / Section | `news-section` |
| 文章标题（H1） | M8 的文章标题（CTR 优化版，含关键词+年份+钩子词） |
| URL Slug | 包含游戏英文名 + 角度关键词，如 `stzb-alternative-2026` |
| Meta Description | M9 的输出（110~150字符，前15字有钩子词） |
| 正文 | M8 AI 生成内容（折扣占位已替换，含对比表格） |
| 内链① | 子站游戏详情页 `/games/[id]`，锚文字：游戏名 |
| 内链② | 主站盒子页 `/boxes/[id]`，锚文字：「点击查看[X折]福利」 |
| 内链③（可选） | 子站 `/games/` 列表页，锚文字：「查看更多[品类]平替手游」 |
| 图片格式 | WebP，单张 ≤200KB，文件名语义化（如 `stzb-alternative-gamename-01.webp`） |

---

## URL Slug 命名规则

格式：`{截流游戏英文缩写}-{角度关键词}-{年份}`

| 截流游戏 | 英文缩写 | 示例 Slug |
|----------|----------|-----------|
| 率土之滨 | stzb | `stzb-alternative-2026` |
| 原神 | yuanshen | `yuanshen-recharge-guide-2026` |
| 文明与征服 | wmyzz | `wmyzz-casual-alternative-2026` |

规则：
- 全部小写，单词间用 `-` 连接
- 不要用拼音全拼（太长），用约定缩写
- 同一游戏不同角度的文章，角度关键词要不同（`alternative` / `recharge` / `casual` / `zero-cost`）

## 方案一：人工发布

**适用：** 日常发布，按字段清单逐项填写

1. 登录后台
2. 内容管理 → 文章列表 → 查看模式切换为「关联网站」→ 选 SITE_ID=8
3. 点击「新增」
4. 按上方字段清单逐项填写
5. 保存前检查：内链是否有效（在新标签页打开确认）
6. 发布

## 方案二：API 自动发布（程序）

**适用：** 批量发布或自动化流水线

```python
import requests

API_BASE = "https://api.your-backend.com"
API_TOKEN = "YOUR_TOKEN"

def publish_article(
    title: str,
    slug: str,
    content: str,
    meta_description: str,
    site_id: int = 8,
    section: str = "news-section",
):
    payload = {
        "title": title,
        "slug": slug,
        "content": content,
        "meta_description": meta_description,
        "site_id": site_id,
        "section": section,
        "status": "published",
    }
    resp = requests.post(
        f"{API_BASE}/articles",
        json=payload,
        headers={"Authorization": f"Bearer {API_TOKEN}"},
        timeout=15,
    )
    resp.raise_for_status()
    article_id = resp.json()["data"]["id"]
    print(f"发布成功：/news/{article_id}")
    return article_id
```

> API 路径和字段名以实际后台接口文档为准。

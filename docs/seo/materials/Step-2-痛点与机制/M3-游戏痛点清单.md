# M3：游戏痛点清单

> **这份材料回答：** 玩家为什么要离开这款游戏？
> **输入：** M1 的 `game_name`；可选追加真实玩家评论
> **输出：** 12~15 条具体痛点，按维度分类
> **后续用途：** M5 角度矩阵的 `pain_points`；M8 文章的 `pain_points`（取 3~5 条）

---

## 输出格式（标准化）

```
## 【时间压力】
1. 每天必须完成12个日常任务，公会战强制在线否则被踢 → 导致上班族、轻度玩家大量流失
2. 活动期间需要连续签到7天一天不缺，断签即失去全部积累 → 导致出差/旅游玩家流失

## 【付费压力】
3. 月卡性价比低，非月卡玩家资源积累速度是月卡的1/5 → 导致零氪/微氪玩家流失
...
```

维度：时间压力 / 付费压力 / 社交绑架 / 运营问题 / 竞技失衡

---

## 方案一：AI 直接生成（适合大热门游戏）

**适用条件：** 游戏知名度高（月活百万+），AI 对游戏机制有真实了解

**验证方法：** 痛点描述中是否出现了该游戏独有的系统名称（如"征服赛季""铺路"）—— 有具体词 = 可信；全是通用描述 = 改用方案二

### 系统提示词

```
你是一名资深手游玩家和游戏社区运营，熟悉国产手游的常见问题和玩家流失规律。

请分析用户给出的游戏，列出玩家最常见的 12~15 个离开理由或强烈痛点。

要求：
1. 每条痛点必须具体，不能泛泛而谈
   - 错误示例："游戏太肝"
   - 正确示例："每天必须完成12个日常任务，公会战强制在线否则被踢，活动期间需要连续签到7天一天不缺"
2. 每条痛点后注明：这个痛点导致哪类玩家流失（例："导致上班族、轻度玩家大量流失"）
3. 按以下维度分组输出：
   - 【时间压力】需要每天固定在线、肝日常等
   - 【付费压力】氪金门槛、月卡/礼包性价比差等
   - 【社交绑架】公会压力、强制组队等
   - 【运营问题】版本更新慢、BUG多、客服差、停服风险等
   - 【竞技失衡】充值玩家碾压、零氪上限低等

输出格式：
## 【维度名称】
1. [具体痛点描述] → 导致[流失用户画像]
```

### 用户提示词

```
游戏名：${game_name}
```

## 方案二：AI + 玩家评论（适合冷门游戏）

**适用条件：** 游戏知名度较低，或 AI 给出的痛点缺乏游戏专属细节时

**评论来源：** TapTap → 搜索游戏 → 评价 → 按「差评」排序 → 复制 10~20 条

### 系统提示词

```
你是一名手游用户研究员，擅长从玩家评论中提取真实的游戏痛点。

请从用户提供的玩家评论中，提取这款游戏的真实离开理由，要求：
1. 合并重复出现的抱怨，每类痛点用一句具体描述（必须包含评论中出现过的具体细节）
2. 标注这条痛点在评论中出现的频率：高/中/低
3. 按维度分类：时间压力 / 付费压力 / 社交绑架 / 运营问题 / 竞技失衡

输出格式：
## 【维度名称】
1. [具体痛点描述（含评论原话关键词）] → 出现频率：高/中/低 → 导致[流失用户画像]
```

### 用户提示词

```
游戏名：${game_name}

玩家评论（从 TapTap/App Store 差评区复制 10~20 条）：
${player_reviews}
```

| 变量 | 含义 | 来源 |
|------|------|------|
| `game_name` | 游戏名 | M1 输出 |
| `player_reviews` | 10~20 条真实差评 | TapTap 差评区复制 |

## 方案三：程序生成

**适用：** 批量处理多款游戏，自动抓取评论并提取痛点

**两步流程：**

**步骤一：程序抓取 TapTap 差评**

```python
import requests
from bs4 import BeautifulSoup

def fetch_taptap_negative_reviews(game_id, page=1):
    """
    抓取 TapTap 某游戏的差评（1~2星）
    game_id: TapTap 游戏 ID，从游戏页 URL 获取
    """
    url = f"https://www.taptap.cn/app/{game_id}/review"
    params = {"score": "1,2", "page": page}  # 筛选差评
    headers = {"User-Agent": "Mozilla/5.0"}
    resp = requests.get(url, params=params, headers=headers, timeout=10)
    soup = BeautifulSoup(resp.text, "html.parser")
    
    reviews = []
    for item in soup.select(".review-item__content"):
        reviews.append(item.get_text(strip=True))
    return reviews

# 使用示例
reviews = fetch_taptap_negative_reviews(game_id="12345", page=1)
reviews += fetch_taptap_negative_reviews(game_id="12345", page=2)
```

**步骤二：把评论文本送给 AI 提取痛点**（使用方案二的提示词）

> TapTap 的 game_id 在游戏页 URL 中：`https://www.taptap.cn/app/{game_id}`

## 方案四：人工整理

**适用：** 你本身是游戏玩家，有第一手体验

1. 打开游戏贴吧 → 搜「弃坑」「删号」「不想玩了」
2. 打开 TapTap 差评区 → 找高赞差评
3. 打开 B站 → 搜「[游戏名] 弃坑」「[游戏名] 劝退」
4. 提炼出 12~15 条，按输出格式填写

---

## 质量检验

生成后检查：

- [ ] 每条痛点是否含有该游戏的具体机制名称或系统名词
- [ ] 每条痛点是否有明确的流失用户画像
- [ ] 至少覆盖 3 个以上维度
- [ ] 没有两条描述的是同一个问题

# M6：今日写作任务

> **这份材料回答：** 今天应该写哪几篇文章？
> **输入：** M5 `task_matrix`（所有游戏的任务卡）+ 已发布去重索引
> **输出：** 今日 3~5 个带优先级的写作任务
> **后续用途：** 每个任务单独进入 M8 生成完整文章

---

## 输出格式（标准化）

```
| 优先级 | 游戏名 | 角度类型 | 目标关键词 | 推荐标题 | 写作重点提示 |
|--------|--------|----------|------------|----------|--------------|
| 1 | 率土之滨 | 截流型 | 类似率土之滨的游戏 | 2026年类似率土之滨的SLG手游... | 重点突出低肝度，首段用「征服赛季」梗开场 |
| 2 | 原神 | 充值避坑型 | 原神充值值不值 | 原神月卡2026年还值得买吗... | 对比月卡与免费玩家的资源差距数据 |
```

去重索引格式（存入素材卡）：
```
率土之滨|截流型
率土之滨|痛点解决型
原神|充值避坑型
```

---

## 方案一：AI 筛选

**适用：** 任务库积累了多款游戏后，让 AI 按优先级规则自动选

### 系统提示词

```
你是一名内容运营，负责从手游攻略站的文章待写任务库中，每日挑选最高价值的任务来生产。

请根据用户提供的任务库和已发布去重索引：
1. 排除已发布的「游戏名×角度类型」组合
2. 按以下优先级，选出今天最值得写的 5 篇：

优先级规则（流量 × 转化 双维度）：
1. 有明确停服/回归/联动事件触发 → 最高优先（时效窗口短，立刻写）
2. 关键词有搜索量 且 为截流型/平替推荐型 → 高优先（流量×转化双赢）
3. 关键词有搜索量 且 为充值避坑型 → 高优先（商业转化意图极强）
4. 痛点解决型（对应关键词在搜索词表中能找到）→ 中优先
5. 入坑决策型 + 零氪攻略型 → 正常优先
6. 无对应搜索词的时间友好型/跑路停服型 → 备选

输出格式（5行表格）：
| 优先级 | 游戏名 | 角度类型 | 目标关键词 | 推荐标题 | 写作重点提示（一句话） |
|--------|--------|----------|------------|----------|------------------------|
```

### 用户提示词

```
待写任务库（各游戏 M5 输出汇总）：
${task_matrix}

已发布去重索引（格式：游戏名|角度类型，每行一条；无则填"暂无"）：
${published_index}

今天的游戏偏向（不限则填"不限"）：${today_preference}
```

| 变量 | 含义 | 来源 |
|------|------|------|
| `task_matrix` | 多款游戏角度矩阵合并 | 复制各游戏 M5 输出表格 |
| `published_index` | 已发布文章的二元组索引 | 从内容追踪矩阵提取 ✅ 条目 |
| `today_preference` | 今天想写的题材方向（可选） | 三国题材 / 不限 |

## 方案二：程序自动筛选

**适用：** 任务库大（50篇以上积累），想完全自动化

**优先级评分规则实现：**

```python
PRIORITY_SCORES = {
    "截流型": 90,
    "平替推荐型": 85,
    "充值避坑型": 85,
    "痛点解决型": 70,
    "入坑决策型": 60,
    "零氪攻略型": 55,
    "时间友好型": 40,
    "跑路停服型": 35,
}

SEARCH_VOLUME_BONUS = {"高": 30, "中": 15, "低": 0, "待查": 5}

def score_task(task, published_index):
    """给单个任务评分"""
    key = f"{task['游戏名']}|{task['角度类型']}"
    if key in published_index:
        return -1  # 已发布，排除
    
    base = PRIORITY_SCORES.get(task["角度类型"], 30)
    volume_bonus = SEARCH_VOLUME_BONUS.get(task.get("搜索量级", "待查"), 5)
    return base + volume_bonus

def select_daily_tasks(all_tasks, published_index, today_preference=None, n=5):
    scored = [(task, score_task(task, published_index)) for task in all_tasks]
    scored = [(t, s) for t, s in scored if s >= 0]  # 过滤已发布
    
    if today_preference:
        # 偏向特定题材的任务
        scored = sorted(scored, key=lambda x: (
            x[0].get("题材") == today_preference, x[1]
        ), reverse=True)
    else:
        scored.sort(key=lambda x: x[1], reverse=True)
    
    return [t for t, _ in scored[:n]]
```

## 方案三：人工选定

**适用：** 任务少（10篇以内积累），直接看表格选

1. 打开 M5 输出的各游戏任务卡
2. 划掉已发布的角度（查内容追踪矩阵）
3. 按优先级规则手动选 3~5 篇
4. 填入今日写作任务表格

## 更新去重索引

每次写完一篇文章，立即在内容追踪矩阵中更新：

```
素材卡「已完成的文章角度」栏：
✅ 截流型 | 2026-04-23
```

下次运行 M6 时，把所有 ✅ 条目提取成 `游戏名|角度类型` 格式粘给 `published_index`。
